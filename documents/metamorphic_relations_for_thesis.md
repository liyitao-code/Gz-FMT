## 蜕变测试关系设计与实现总览

本文档总结当前在 `randomsmith_meta.py` 中已经实现、并在实验中实际使用的蜕变测试关系。  
每个小节依次描述：

- **内容与直观含义**
- **测试目的 / 针对的潜在 bug 类型**
- **设计思路与实验流程**
- **变换逻辑与形式化蜕变关系**
- **结果与判定方法（包含主要阈值）**
- **当前实验中是否发现可疑问题**

符号约定：  
- 世界 \(W\)，模型 \(M\)；  
- 位置向量 \(p(t) = (x(t), y(t), z(t))\)；  
- 位移 \(\Delta p = p(t_2)-p(t_1)\)，其范数记为 \(\|\Delta p\|\)；  
- 力向量 \(F = (F_x, F_y, F_z)\)，质量 \(m\)，仿真时间 \(T\)。

## 范式总览：四种范式及其对应的蜕变关系

从**实验结构**与**比较对象**出发，本工作中的蜕变关系可归为以下四类范式。理解范式有助于统一描述“一次实验里跑几次、比什么”。

| 范式 | 英文名（便于论文引用） | 含义 | 典型问题 |
|------|------------------------|------|----------|
| **A** | **Multi-Run Compare** | **多次独立运行**（可不同进程、不同环境或不同参数），对**同一逻辑输入**比较输出是否一致或满足理论关系。 | 两次运行终点是否一致？Reset 后行为是否与 fresh start 一致？ |
| **B** | **Single-Run Invariant** | **单次运行**内，在多个时间点或条件下检查某个**不变量**是否成立（不依赖 reset、不比较两次运行）。 | 无力时是否静止？位移是否单调？两 link 距离是否稳定？ |
| **C** | **Same-Run Multi-Model** | **单次运行**内，对**多个模型**施加不同干预（如只对其中一个施力），比较它们的行为是否满足隔离/独立性。 | 只对 A 施力时，B 是否不动？ |
| **D** | **Sequential No-Reset** | **单次运行**内，按时间分为**连续阶段**（如先施力再撤力、先质量 m 再改质量 k·m），**不 reset**，比较各阶段的量是否满足理论关系。 | 撤力后速度是否不再增加？改质量后位移比是否符合 F=ma？ |

### 各范式对应的蜕变关系一览

- **范式 A（Multi-Run Compare）**  
  - **Run-level Determinism（双进程确定性）**：两进程、同 SDF/同脚本/同 seed → 比较终点位姿。  
  - **Rewind / Playback 一致性**：记录脚本重放 → 与原始运行比较。  
  - **Reset 等价性**：fresh start 与 reset 后继续运行 → 行为应等价（如浮力仍生效）。  
  - **环境扰动鲁棒性**：Phase A 无装饰物、Phase B 插入远处静态模型（需 reset 后做 B）→ 比较同一模型位移。  
  - **时间缩放**：rtf=1.0 与 rtf=r 两次运行（中间 reset）→ 比较同物理时间位移。  
  - **质量缩放（带 Reset）**：质量 m 跑一次、reset 后改为 k·m 再跑一次 → 比较位移比 d₁/d₂≈k。  
  - **力的可加性**：F₁+F₂ 跑一次、reset 后 F_tot 跑一次 → 比较位移。  
  - **对称性**：沿轴 a 施力跑一次、reset 后沿轴 b 施力跑一次 → 比较两轴位移大小。

- **范式 B（Single-Run Invariant）**  
  - **零输入稳定性**：单次运行、不施力，检查水平漂移是否 < ε。  
  - **时序单调性**：单次运行、恒定力，检查位移序列是否单调且平滑。  
  - **关节约束稳定性**：单次运行、施力，检查两 link 距离是否无漂移/无跳变。

- **范式 C（Same-Run Multi-Model）**  
  - **多模型力隔离**：单次运行、只对 target 施力，检查 bystander 是否几乎不动。

- **范式 D（Sequential No-Reset）**  
  - **撤力响应**：单次运行、先施力再撤力滑行，检查撤力后速度变化趋势。  
  - **质量缩放无 Reset**：单次运行、Phase A 质量 m 施力 F，Phase B 改质量 k·m 施力 k·F，比较 d₁ 与 d₂。

另外两条与** API 语义**相关、不按“跑几次”划分，但可理解为对**单次操作**的不变量假设：

- **暂停状态下 set_pose 的速度保持**：在 pause 下调用 set_pose 后，速度应保持不变（为其他需要“恢复位姿不恢复速度”的测试提供前提）。  
- **Reset 等价性**（见上）同时也可视为对“reset 操作”的语义约束，归在范式 A 的 Multi-Run Compare 中。

## 一、零输入稳定性（Zero-Input Stability, 范式 B）

### 1. 内容与目的

- **内容**：在不对模型施加任何外力的情况下，仅推进仿真一段时间，检查模型在水平方向是否基本保持静止。  
- **目的**：发现“无原因的漂移”，例如：
  - 幽灵力 / 数值噪声积累；
  - 插件初始化错误带来的隐藏力或初始速度；
  - 重力或接触求解中的能量注入问题。

### 2. 设计思路与流程

对随机选择的可测试模型 \(M\)：

1. 通过 `scene/info` 获取初始位置 \(p_0 = (x_0, y_0, z_0)\)。
2. 暂停仿真，确保当前状态一致。
3. **不施加任何外力与扭矩**，仅通过 `multi_step` 推进 \(T\) 秒（通常 5 s 左右）。
4. 通过 `scene/info` 获取最终位置 \(p_T = (x_T, y_T, z_T)\)。
5. 计算水平漂移：
   \[
   \Delta x = x_T - x_0,\quad \Delta y = y_T - y_0,\quad
   d_h = \sqrt{\Delta x^2 + \Delta y^2}
   \]

### 3. 变换逻辑与形式化描述

该测试属于**单次运行不变量型蜕变关系**（范式 B），不是“输入变换”，而是对**同一输入下结果满足的必然关系**：

- 条件：外力 \(F \equiv 0\)，除重力与约束外，无显式外力；
- 结论：水平方向的位移范数应在一个很小的容差内。

形式化地，对零输入稳定的系统期望：
\[
F_{\text{ext}}(t) = 0 \Rightarrow
\left\| (x_T - x_0,\, y_T - y_0) \right\| < \varepsilon_h
\]
实现中使用 \(\varepsilon_h = 0.05\ \text{m}\)（5 cm）。

### 4. 判定方法

- **通过条件**：
  \[
  d_h < 0.05\ \text{m}
  \]
- **失败条件**：\(d_h \ge 0.05\ \text{m}\)，会在结果中记录初末位置与各轴漂移。

### 5. 实验发现概况

- 在 `test_22` 中，大量 world/model 组合被检测出“显著漂移”，其中一部分来自：
  - 地面有坡度、重力方向不垂直地面等**场景本身预期会运动**的情况；
  - 另一部分则更像数值或实现问题。  
- 对前一种情况，已经整理为 `zero_input_stability_exclusions.md` 中的**排除列表**，后续测试时不再对这些 pair 应用零输入稳定性测试。  
- 因此，该蜕变关系**确实能暴露异常行为**，但需要结合场景语义（是否“应该静止”）进行过滤。

## 二、多模型力隔离（Force Isolation, 范式 C）

### 1. 内容与目的

- **内容**：在同一 world 中选择两个相距较远的模型 \(M_\text{target}, M_\text{bystander}\)，只对 target 施加水平力，检查 bystander 是否保持静止。  
- **目的**：检测：
  - 力路由错误（例如错误地把力施加到所有实体）；
  - 插件使用全局变量或错误的实体索引，导致**力泄漏**；
  - 广义力实现中“按顺序遍历实体”而非按 id 绑定的 bug。

### 2. 设计思路与流程

1. 在可测试模型集中，找到一对满足水平距离 \(> 2\ \text{m}\) 的模型，记为 \((M_t, M_b)\)。  
2. 记录二者初始位置 \(p_t^0, p_b^0\)。  
3. 暂停仿真，只对 \(M_t\) 施加一个足够大的水平力 \(F = (F_x,0,0)\)。  
4. 使用 `multi_step` 推进 \(T\) 秒。  
5. 记录最终位置 \(p_t^T, p_b^T\)，计算：
   - 目标位移 \(d_t = \|(p_t^T - p_t^0)_{xy}\|\)；
   - 旁观者漂移 \(d_b = \|(p_b^T - p_b^0)_{xy}\|\)。

### 3. 形式化蜕变关系

在理想的隔离系统中：
\[
F \text{ 仅作用于 } M_t \Rightarrow
\|(p_b^T - p_b^0)_{xy}\| < \varepsilon_b
\]
实现中：
- 选择 \(T\) 与 \(F\) 使得 \(d_t\) 明显大于 0（目标必须确实动起来）；
- 设置旁观者容差 \(\varepsilon_b\) 为较小值（代码中基于经验阈值，量纲为米）。

### 4. 判定方法

- 要求：
  - \(d_t\) 大于最小位移（例如 \(> 0.1\ \text{m}\)），以排除被刚性约束住的目标；
  - \(d_b\) 小于容差（例如若达到厘米级以上则报告为可疑）。  
- 报告中同时给出 `Target Displacement` 与 `Bystander Drift`，便于对比。

### 5. 实验发现概况

- 在随机 world 中，该测试的 FAIL 案例相对**不多**，但一旦出现，一般意味着比较严重的建模或插件问题（旁观者模型“被带着走”）。  
- 目前尚未对所有 FAIL 个案做系统 issue 提交，但该关系已验证可以捕捉到“跨模型耦合异常”的现象。

## 三、撤力响应（Force Removal Response, 范式 D）

### 1. 内容与目的

- **内容**：对模型施加一段时间的持续力，然后清除力，观察撤力后速度变化是否合理。  
- **目的**：检查：
  - `clear_model_wrench` 是否真正清除了持续力；
  - 是否存在“力清除后仍继续加速”的能量泄漏；
  - 插件中对力持久化或清除逻辑的错误。

### 2. 设计思路与流程

1. 选取模型 \(M\)，获取初始位置 \(p_0\)。  
2. **Phase 1（施力阶段）**：  
   - 对 \(M\) 施加恒定力 \(F=(F_x,0,0)\) 持续 \(T_\text{force}\)；  
   - 推进仿真，得到位置 \(p_\text{force}\)。  
3. **Phase 2（撤力滑行阶段）**：  
   - 调用 `clear_model_wrench` 撤力；  
   - 将后续滑行时间 \(T_\text{coast}\) 分为两半：前半得到 \(p_{\text{coast},1}\)，后半得到 \(p_{\text{coast},2}\)。  
4. 估算一维速度（沿 x 方向）：
   \[
   v_\text{force} = \frac{x_\text{force} - x_0}{T_\text{force}},\quad
   v_{\text{coast},1} = \frac{x_{\text{coast},1} - x_\text{force}}{T_\text{coast}/2},\quad
   v_{\text{coast},2} = \frac{x_{\text{coast},2} - x_{\text{coast},1}}{T_\text{coast}/2}
   \]

### 3. 形式化蜕变关系

从牛顿第二定律出发，施力阶段结束、撤力后，若只有阻力（摩擦、阻尼等），则沿力方向加速度应不再为正。离散近似下：
\[
F_\text{persistent} \text{ 已清除} \Rightarrow
v_{\text{coast},2} \le v_{\text{coast},1} \le v_\text{force} + \delta_v
\]
其中 \(\delta_v\) 为数值容差。

### 4. 判定方法

实现中大致检查：

- 滑行速度 \(v_{\text{coast},1}, v_{\text{coast},2}\) 不应显著大于施力阶段平均速度 \(v_\text{force}\)；  
- 若撤力后速度仍持续增加，或二段滑行之间速度显著上升，则判为 FAIL。

### 5. 实验发现概况

- 在目前的随机测试规模下，此测试尚未出现大量稳定 FAIL，但它为**验证力清除语义是否正确**提供了一个系统化手段，可在后续扩大样本时继续使用。

## 四、时序单调性（Temporal Monotonicity, 范式 B）

### 1. 内容与目的

- **内容**：在恒定力作用下，以固定时间间隔采样模型在力方向上的位移，检查是否随时间单调递增且相对平滑。  
- **目的**：发现：
  - 刚体求解器的数值不稳定（位移回退、抖动）；
  - 接触/碰撞处理中的“瞬移”或穿透纠正异常；
  - 力间歇性失效（例如控制器周期性关闭力）。

### 2. 设计思路与流程

1. 对模型 \(M\) 获取初始位置 \(x_0\)。  
2. 沿 \(+x\) 方向施加恒定力 \(F\)，总测试时间 \(T\)，采样点数 \(K\)。  
3. 按时间步长 \(\Delta t = T/K\) 推进仿真，并在每个采样点记录 \(x(t_i)\)，得到序列：
   \[
   \{(t_i, x_i)\}_{i=0}^K,\quad x_i = x(t_i)
   \]
4. 计算位移 \(d_i = x_i - x_0\)。

### 3. 形式化蜕变关系

理想情形下（忽略接触与饱和）：
\[
\forall i>0,\ d_i > d_{i-1}
\]
实现中允许小量数值噪声，但要求：

- **单调性**：
  \[
  d_i > d_{i-1} \quad \text{（否则记为 non\_monotonic 违规）}
  \]
- **平滑性**：增量 \(\Delta d_i = d_i - d_{i-1}\) 不应突然放大数倍：
  \[
  \frac{\Delta d_i}{\Delta d_{i-1}} \le 3.0
  \quad\text{（当 } \Delta d_{i-1}>10^{-3}\text{ 时）}
  \]

### 4. 判定方法

- 若最终位移 \(|d_K|\) 过小（例如 < 0.01 m），认为模型受约束，测试跳过；  
- 否则：
  - 有任意 \(d_i \le d_{i-1}\) → 单调性失败；  
  - 有任意 \(\frac{\Delta d_i}{\Delta d_{i-1}} > 3\) → 平滑性失败。  
- 同时把违例点（时间、位移）记录在 `violations` 中。

### 5. 实验发现概况

- 在大量 world 中，该测试可用于筛查“偶发回跳”或“瞬移”的轨迹；  
- 已有若干世界被发现存在明显 non-monotonic 片段，用于后续手工复现与 issue 编写。

## 五、对称性 / 各向同性（Symmetry / Isotropy, 范式 A）

### 1. 内容与目的

- **内容**：在两个重力分量相同的坐标轴上（通常为 x 和 y），对同一模型施加同样大小的力，比较两方向位移的大小是否一致。  
- **目的**：检测：
  - 插件或控制逻辑中**硬编码某一轴**导致行为不对称；
  - 碰撞检测或惯性张量计算在特定方向上的偏差；
  - 坐标系变换实现错误（如只在一个轴上应用旋转）。

### 2. 设计思路与流程

1. 从 SDF 中读取重力向量 \(g = (g_x,g_y,g_z)\)，选择两轴 \(a,b\) 满足 \(g_a \approx g_b\)（优先选两者均约为 0）。  
2. 对模型 \(M\) 获取初始位置 \(p_0\)。  
3. **Test A**：沿轴 \(a\) 施加力 \(F_a = F\mathbf{e}_a\)，运行 \(T\) 秒，得到 \(p_a\)，计算位移分量 \(d_a\)。  
4. Reset 后保持同样初始状态。  
5. **Test B**：沿轴 \(b\) 施加力 \(F_b = F\mathbf{e}_b\)，运行 \(T\) 秒，得到 \(p_b\)，计算 \(d_b\)。

### 3. 形式化蜕变关系

在重力对两轴影响相同的情形下，物理空间应具有各向同性：
\[
F\mathbf{e}_a,\ F\mathbf{e}_b \Rightarrow |d_a| \approx |d_b|
\]
实现中给出相对误差容差 \(\varepsilon_\text{iso}\)，例如：
\[
\frac{\big|\,|d_a| - |d_b|\,\big|}{\max(|d_a|,|d_b|)} < \varepsilon_\text{iso}
\]

### 4. 实验发现概况

- 目前尚未在大规模实验中观察到非常极端的对称性破坏，但该测试为**针对“某个方向明显更粘/更滑”**的问题提供了自动化检查手段。

## 六、力的可加性（Force Additivity, 范式 A）

### 1. 内容与目的

- **内容**：比较“同时施加两个力 \(F_1, F_2\)”与“施加合力 \(F_\text{tot}=F_1+F_2\)”得到的最终位移是否一致。  
- **目的**：检查：
  - 力叠加实现是否正确（是否线性相加）；
  - 持续力与瞬时力的组合是否被错误处理；
  - 插件对力的索引/缓存是否错误（导致只应用最后一个力等）。

### 2. 设计思路与流程

1. 选择模型 \(M\)，记录初始位置 \(p_0\)。  
2. 生成两个与质量成比例的随机力 \(F_1, F_2\)，计算 \(F_\text{tot}=F_1+F_2\)。  
3. **Test A**：在暂停状态下连续发送 `wrench/persistent` 消息添加 \(F_1\) 与 \(F_2\)，推进 \(T\) 秒，得到 \(p_{F_1+F_2}\)。  
4. Reset 仿真，恢复到初始位姿。  
5. **Test B**：只施加 \(F_\text{tot}\)，同样推进 \(T\) 秒，得到 \(p_{F_\text{tot}}\)。  
6. 比较两次位移向量。

### 3. 形式化蜕变关系

线性系统中有：
\[
\Phi(F_1 + F_2) = \Phi(F_1) + \Phi(F_2)
\]
在这里用位移来近似 \(\Phi\)，测试关系化为：
\[
\Delta p(F_1,F_2) \approx \Delta p(F_\text{tot}),\quad
\Delta p(F_1,F_2) = p_{F_1+F_2}-p_0,\ 
\Delta p(F_\text{tot}) = p_{F_\text{tot}}-p_0
\]
实现中用欧氏距离与相对误差作为判据。

### 4. 实验发现概况

- 在有限样本上，该测试尚未暴露出明显的“只吃最后一个力”之类的错误，但从理论上对复杂插件/控制器仍有较强覆盖力。

## 七、时间缩放（Time Scaling, 范式 A）

### 1. 内容与目的

- **内容**：对同一模型施加相同力 \(F\)，比较 `real_time_factor=1.0` 与其它值（如 0.5〜2.0）下，**在相同物理时间 \(T\)** 的最终位置是否一致。  
- **目的**：验证：
  - `real_time_factor` 只影响**仿真速度与 wall-clock 对应关系**，而不应改变物理结果；
  - time step 自适应或控制逻辑在变速时是否引入物理偏差。

### 2. 设计思路与流程

1. 在 rtf=1.0 下，对模型 \(M\) 施加力 \(F\) 持续 \(T\)，得到位移 \(\Delta p_1\)。  
2. Reset 仿真并恢复初始位姿。  
3. 设置 rtf=\(r_2\in[0.5,2.0]\)，施加**完全相同**的力 \(F\) 持续同样的物理时间 \(T\)，得到 \(\Delta p_2\)。  
4. 比较 \(\Delta p_1\) 与 \(\Delta p_2\)。

### 3. 形式化蜕变关系

设仿真算子 \(\Phi(W,M,F,T;\text{rtf})\) 给出物理结果，则期望：
\[
\Phi(W,M,F,T;\text{rtf}=1.0) \approx \Phi(W,M,F,T;\text{rtf}=r_2)
\]
对位移向量有：
\[
\|\Delta p_1 - \Delta p_2\| < \varepsilon_\text{rtf}
\]

### 4. 实验发现概况

- 初步实验中，并未在主流场景里观察到极端时间缩放 bug；  
- 该测试更适合作为**回归测试**，防止今后对 scheduler/stepper 的修改破坏 `rtf` 语义。

## 八、质量缩放（Mass Scaling, 两种变体）

### 1. 带 Reset 的质量缩放（`metamorphic_test_mass_scaling`, 范式 A）

#### 1.1 内容与目的

- **内容**：比较质量为 \(m\) 与质量为 \(k m\) 时，在相同力 \(F\) 下的位移比值是否符合 F=ma。  
- **目的**：检测：
  - 质量在动力学方程中的作用是否正确实现；
  - 修改 SDF 质量后，求解器/插件是否真正生效；
  - 控制器是否硬编码了质量或使用了错误的质量值。

#### 1.2 形式化蜕变关系

在理想的匀加速运动中：
\[
a = \frac{F}{m},\quad s = \frac{1}{2} a T^2
\]
因此，若 \(m_2 = k m_1\)，在相同 \(F,T\) 下，有：
\[
\frac{d_1}{d_2} \approx k
\]
其中 \(d_i\) 为对应质量下的位移模长。

实现中使用相对误差阈值：
\[
\left|\frac{d_1}{d_2} - k\right| \Big/ k < \varepsilon_m
\]

### 2. 无 Reset 的质量缩放（`metamorphic_test_mass_scaling_no_reset`, 范式 D）

#### 2.1 内容与目的

- **内容**：在同一运行中分两个阶段：
  - **Phase A**：质量 \(m\)，力 \(F\)，位移 \(d_1\)；
  - **Phase B**：质量 \(k m\)，力 \(k F\)，位移 \(d_2\)；
  检查 \(d_1\) 与 \(d_2\) 是否接近。  
- **目的**：在更接近真实使用场景（运行中动态改质量）下，检测质量相关 bug。

#### 2.2 形式化蜕变关系

在两阶段中保持加速度 \(a = F/m\) 不变，因此理论上：
\[
d_1 \approx d_2
\]
实现中使用相对误差：
\[
\frac{|d_1 - d_2|}{\max(d_1,d_2)} < \varepsilon_\text{rel}
\]
默认 \(\varepsilon_\text{rel}=0.25\)，并额外约束 \(d_2/d_1\) 在合理范围 \([0.3,3.3]\) 内。

#### 2.3 实验发现概况（test_23）

- 在 `test_23` 中，`mass_scaling_no_reset` 的完成用例里 **FAIL 占比较高**（约 9/15），主要集中在：
  - 带差速驱动或控制器的车辆（`vehicle`, `vehicle_green`, `vehicle_blue` 等）；
  - 多关节摆类模型。  
- 这些 FAIL 既可能暴露真实的质量处理问题，也反映了**控制器回路 + 质量修改**带来的复杂行为，目前适合作为“可疑样本”重点分析。

## 九、环境扰动鲁棒性（Environment Perturbation Robustness, 范式 A）

### 1. 内容与目的

- **内容**：在同一 world 中，分别在“无装饰物”和“插入远处静态装饰物”两种环境下对同一模型施加相同的力，比较位移差异。  
- **目的**：检测：
  - 力或约束实现中错误地遍历/使用“所有实体”，包括本不相关的装饰物；
  - 插件使用全局状态、依赖实体遍历顺序或 world 索引的 bug；
  - 全局场（如流体、风场）被错误地作用到所有物体。

### 2. 设计思路与流程

1. 选择 world 中一个可移动模型 \(M\)，记录初始位置 \(p_0\)，质量 \(m\)。  
2. 计算水平力 \(F=(F_x,0,0)\)，使其足够大以产生显著位移。  
3. **Phase A**（无装饰物）：
   - 在当前场景下对 \(M\) 施加 \(F\) 持续 \(T\)，得到位移向量 \(\Delta p_A\)，模长 \(d_A\)。  
   - 若 \(d_A\) 过小、过大（>50m）、或落入异常 z 高度，则跳过此 world。  
4. Reset world。  
5. **Phase B**（插入装饰物）：
   - 在距离 \(M\) 水平约 15 m 处插入静态装饰模型（如 box）；  
   - 再次对 \(M\) 施加相同 \(F\) 与 \(T\)，得到 \(\Delta p_B\)，模长 \(d_B\)。  
6. 计算相对误差：
   \[
   \text{rel\_err} = \frac{\|\Delta p_A - \Delta p_B\|}{\max(d_A,d_B,10^{-9})}
   \]

### 3. 形式化蜕变关系

环境只改变“远处静态装饰物”，不改变重力、接触地面或外加载力，因此理论上：
\[
\frac{\|\Delta p_A - \Delta p_B\|}{\max(d_A,d_B)} < \varepsilon_\text{env}
\]
实现中默认 \(\varepsilon_\text{env}=0.15\)。

### 4. 实验发现概况（test_23）

- `environment_perturbation_robustness` 在 `test_23` 中完成的用例里，有 **6 个 FAIL**，涵盖：
  - 单/双摆（`pendulum2` 等），位移相对误差可达 20%〜150%；  
  - 车辆（`vehicle_green`），在插入装饰物后位移有 20% 以上差异。  
- 这些结果说明：**在部分 world 中，远处静态实体的存在会显著改变目标模型的运动**，值得深入排查是否为实现 bug。

## 十、关节约束稳定性（Joint Constraint Stability, 范式 B）

### 1. 内容与目的

- **内容**：对具有多个 link 的模型，选取某个关节连接的两 link，检查它们之间的“约束距离”在施力扰动下是否保持稳定。  
- **目的**：检测：
  - 约束求解器中的能量爆炸或数值漂移；
  - 关节实现错误（如错误的锚点、长度或约束方向）；
  - step / reset 时约束状态不一致导致的“跳变”。

### 2. 设计思路与流程

1. 从场景中筛选出**多 link 模型**，并通过 `pose/info` 解析 link 位姿，得到可用 link 对 \((L_1,L_2)\)。  
2. 使用 `pose/info` 获取初始距离：
   \[
   d_0 = \|p_{L_1}(0) - p_{L_2}(0)\|
   \]
   若 \(d_0\) 过大（>50m）或过小，认为 link 解析错误，跳过。  
3. 根据模型质量生成水平+侧向组合力 \(F\)，对整个模型施加（通过某个刚体 link），模拟实际受扰动情形。  
4. 在总时长 \(T\) 内，分 \(K\) 次推进，每次推进后通过 `pose/info` 读取 link 位姿，记录：
   \[
   d_i = \|p_{L_1}(t_i) - p_{L_2}(t_i)\|,\quad i=1..K
   \]

### 3. 形式化蜕变关系

对稳健的约束，应满足：

- **无漂移**：
  \[
  \forall i,\ |d_i - d_0| < \varepsilon_\text{drift}
  \]
- **无跳变**：
  \[
  \forall i\ge2,\ |d_i - d_{i-1}| < \varepsilon_\text{jump}
  \]

实现中默认：
- \(\varepsilon_\text{drift}=0.05\ \text{m}\)；
- \(\varepsilon_\text{jump}=0.1\ \text{m}\)。

### 4. 判定方法与实验发现（test_23）

- 若所有采样点均满足上述两个条件，则测试通过；否则记录所有违规点（时间、距离、漂移/跳变量）。  
- 在 `test_23` 中：
  - `vehicle`（前轮与车身）在少数轮次出现边缘漂移/跳变（约 6〜10 cm），属于**轻微稳定性问题或阈值边界**；  
  - `cylinder` 模型在多次测试中出现**极端距离爆炸**（从 1.5 m 增长到数百〜数万米），明显表明该 world 下的约束/求解存在严重问题或 link 选取不适配。  
- 此外，对 `_5` 用例的复现实验表明：**原始 FAIL 在当前环境下不可稳定复现**，说明这类边缘用例还受版本/流程影响，需要在论文中如实说明“不可复现的历史 FAIL”这一类情况。

## 十一、确定性与回放（Determinism & Rewind, 范式 A）

虽然在代码中 Determinism 与 Rewind 使用的函数名并非 `metamorphic_test_*` 前缀，但本质上也属于蜕变关系：**同一输入在重复运行或回放时应给出相同输出**。

### 1. Run-level Determinism（双进程确定性）

#### 1.1 内容与目的

- **内容**：对同一 SDF/world，在两个独立的 Gazebo 进程中执行完全相同的施力+步进脚本，比较最终位置 \(P_1,P_2\)。  
- **目的**：检测：
  - 仿真器在相同初态与输入下是否具有**运行间确定性**；
  - 线程调度、浮点非确定性、未初始化状态等导致的 run-to-run 差异。

#### 1.2 形式化蜕变关系

设算子 \(\Phi(W,\text{script};\text{seed})\) 返回最终状态，则期望：
\[
\Phi(W,\text{script};\text{seed}) \approx \Phi(W,\text{script};\text{seed})
\]
在实现中采用两进程 + 同 seed 的方式，比较 \(P_1,P_2\) 的距离：
\[
\|P_1 - P_2\| < \varepsilon_\text{det}
\]

#### 1.3 实验误差与改进

- 目前的实现中，激活过程采用 `pause:false → pause:true`，其间实际运行时间依赖 wall-clock 与调度，导致两轮的“施力前初态”并不完全一致，是一个**已知实验误差来源**。  
- `determinism_replay_operations_and_error.md` 中对这一点做了详细分析，并建议用固定步数激活或记录施力前状态，以减少误差。  
- 实验中观察到的 \(0.3\sim 1.7\ \text{m}\) 的差异，不能简单等价为“仿真器非确定性”，需要在论文中区分“实验流程误差”与“真实非确定性”。

### 2. Rewind / Playback 一致性

- **内容**：对记录好的 `experiment_log.json`，通过外部脚本严格重放所有 `gz service` / `gz topic` 调用，检查重放结果是否与原始实验一致。  
- **目的**：验证：
  - 记录的指令序列是否足以重构实验；
  - 仿真器是否对相同脚本执行给出一致响应；
  - 为 issue 复现与回归测试提供一个可重复的“证据回放”机制。

形式化上，这与 Determinism 十分接近，只是“第二次运行”改为“由脚本驱动的回放”，蜕变关系同样是：
\[
\Phi(W,\text{script}_\text{record};\text{seed}) \approx \Phi(W,\text{script}_\text{replay};\text{seed})
\]

## 十二、Reset 等价性（Reset vs Fresh Start, 范式 A）

这一节对应你提交并已被认可的 issue：**“Buoyancy module failed after resetting the simulation”**。

### 1. 内容与目的

- **内容**：比较“从 SDF 直接启动 world”与“运行一段时间后执行 `reset: {all: true}` 再继续运行”的行为是否等价。  
- **目的**：检测：
  - 世界重置（特别是 `reset: {all: true}`）是否真正恢复到“初始状态”；
  - 插件在 reset 之后是否还能重新发现并正确管理实体（例如浮力插件重新创建组件）；
  - 是否存在依赖 `EachNew` / 动态组件的系统在 reset 之后被“静默失效”的情况。

### 2. 设计思路与流程

以一个包含浮力系统（如 `gz-sim-buoyancy-system`）的 world 为例：

1. **Run A（fresh start）**：  
   - 从 SDF 启动仿真，启动后立即 `pause:false` 运行 \(T\) 秒；  
   - 观察模型（如 `my_turtle`）是否按预期浮在水面上，记录关键状态（如是否存在 `Volume`、`CenterOfVolume` 组件、浮力效果等）。
2. **Run B（reset-then-run）**：  
   - 同样从 SDF 启动；  
   - 运行一段时间后执行：
     - `pause:true`  
     - `reset: {all: true}, pause: true`  
     - 再 `pause:false` 运行同样的 \(T\) 秒；  
   - 观察相同模型是否仍能浮起，检查同一组组件和物理行为是否恢复。

在理想实现中，**Run A 和 Run B 的后续行为应该是等价的**。

### 3. 形式化蜕变关系

设 \(\Phi_\text{fresh}(W,T)\) 表示“从 SDF 直接启动后运行 T 秒”的结果（例如浮力是否生效），  
\(\Phi_\text{reset}(W,T)\) 表示“启动 → reset:all → 再运行 T 秒”的结果，则期望：
\[
\Phi_\text{fresh}(W,T) \approx \Phi_\text{reset}(W,T)
\]
对浮力世界而言，具体到模型 \(M\) 的高度 \(z_M(t)\) 与浮力组件存在性：
\[
\text{浮力正常的充要条件在 Run A 和 Run B 中应同时成立}
\]

### 4. 实验发现与 issue 对应

- 在 `buoyant_cylinder` world 中，你观察到：
  - Run A：模型 `my_turtle` 能稳定浮在水面（z≈0），浮力组件（`Volume` / `CenterOfVolume`）在 ECM 中存在；  
  - Run B：在 `reset: {all: true}` 之后，模型会直接下沉，浮力完全消失。  
- 通过源码分析，你指出：
  - `ResetTo(initialEntityCompMgr)` 把 ECM 还原到**插件第一次 `PreUpdate` 之前**的快照，导致浮力插件新建的组件全部消失；  
  - 重新加载的浮力插件依赖 `EachNew<Link,Inertial>` 来发现实体，但 reset 后这些实体并不视为“新实体”，因此不再被重新标记，浮力计算循环找不到任何 link。  
- 这正是 **Reset 等价性蜕变关系** 的一个反例，并已在官方仓库中作为 bug 被确认。

## 十三、暂停状态下 set_pose 的速度保持（API 语义型）

这一节对应 issue：**“Model velocity not preserved after set_pose during paused simulation”**。不按“跑几次”划分，属于对单次 API 调用的语义不变量假设。

### 1. 内容与目的

- **内容**：在仿真**暂停（pause:true）**状态下，通过 `/world/<world>/set_pose` 修改模型位姿，检查其线速度 / 角速度是否被意外清零或改变。  
- **目的**：验证 Gazebo 提供的 `set_pose` API 是否满足“**只改姿态，不乱改速度**”这一语义假设；这一假设在多个蜕变测试中被当作前提使用（例如重置后用 `set_model_pose` 恢复到初始位姿，再继续施力测试）。

### 2. 设计思路与流程

对一个在运行中的 world：

1. 先让模型 \(M\) 运行一段时间，使其当前速度 \((v,\omega)\) 非零；  
2. 发送 `pause:true`，等待确保仿真暂停；  
3. 记录此时的线速度和角速度：
   \[
   v_\text{before},\ \omega_\text{before}
   \]
4. 调用 `set_pose` 将模型位姿设置为某个新位置 / 朝向 \(p'\)，**不显式更改速度**；  
5. （仍保持 pause 状态或只迈出极短时间步）再次读取速度：
   \[
   v_\text{after},\ \omega_\text{after}
   \]

### 3. 形式化蜕变关系

在 pause 状态下，`set_pose` 预期是一个“纯几何操作”，不应修改动力学量。蜕变关系可写为：
\[
\text{pause} \land (v_\text{before},\omega_\text{before}) = (v,\omega)
\xRightarrow{\text{set\_pose}(p')}
(v_\text{after},\omega_\text{after}) \approx (v,\omega)
\]
在数值实现中允许极小误差，但不允许出现“将非零速度清零”或“产生额外速度”的情形。

### 4. 实验与 issue 对应

- 你在调试使用 `set_model_pose` / `restore_models_state` 的蜕变测试（如时间缩放、质量缩放、force additivity 等）时，发现：
  - 在 pause 状态下调用 `set_pose` 后，模型的速度并未按预期保持，而是被清零或发生跳变；  
  - 这会直接破坏后续基于“同一初态再施力”的蜕变关系（例如时间缩放、质量缩放等），因为第二阶段的真实初态与第一阶段不同。  
- 因此，你将这一 API 语义偏差抽象成上面的蜕变关系，并以 **“Model velocity not preserved after set_pose during paused simulation”** 提交 issue，指出：
  - `set_pose` 目前更像是“**完全重置状态**”而非“只改 pose”；  
  - 对需要高精度控制初始条件的测试和应用（包括本文的多个蜕变关系）会产生系统性误差。

## 十四、小结：蜕变关系体系与论文写作建议

1. **范式层次**（详见上文 **「范式总览」**）  
   - **范式 A（Multi-Run Compare）**：双进程确定性、回放一致性、Reset 等价性、环境扰动鲁棒性、时间缩放、质量缩放（带 reset）、力的可加性、对称性；  
   - **范式 B（Single-Run Invariant）**：零输入稳定性、时序单调性、关节约束稳定性；  
   - **范式 C（Same-Run Multi-Model）**：多模型力隔离；  
   - **范式 D（Sequential No-Reset）**：撤力响应、质量缩放无 Reset。  
   另有两条 **API 语义型** 关系：Reset 等价性（也可归入 A）、暂停下 set_pose 速度保持。

2. **覆盖的物理与实现层面**  
   - 力与质量（F=ma、叠加、撤除）；
   - 时间与 step（rtf、单调性、回放）；
   - 约束与关节（零输入稳定、关节约束稳定性）；
   - 场景与多模型交互（力隔离、环境扰动）。

3. **bug 发现情况**  
   - 在 `test_22`、`test_23` 等批量实验中，**零输入稳定性、环境扰动鲁棒性、质量缩放无 Reset、关节约束稳定性** 已经产生了若干高价值 FAIL，用于后续人工复现与 issue 编写；  
   - 其他关系（如对称性、时间缩放、力可加性、撤力响应）目前更多用于**防回归与结构性保障**，尚未出现大量稳定 FAIL，但在理论上对对应维度具有良好覆盖。

在论文中，可以将以上每个测试视为一个形式化的蜕变关系，对应一个或多个物理/实现假设，并用上述符号化形式给出精确定义，再结合实验章节中的 `test_22` / `test_23` 结果，分别展示其发现的问题与局限性。

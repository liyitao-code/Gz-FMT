# Determinism 复现：具体操作顺序与实验误差分析

## 1. 复现脚本中的完整操作顺序

### Run 1（第一个 Gazebo 进程）

| 序号 | 操作 | 说明 |
|------|------|------|
| 1 | `Popen("gz sim a.sdf")` | 启动进程 1 |
| 2 | `time.sleep(2)` | 壁钟等 2s，等 Gazebo 就绪 |
| 3 | `gz service ... --req 'pause: false'` | **激活：开始仿真** |
| 4 | `gz service ... --req 'pause: true'` | **激活：立即暂停** |
| 5 | `time.sleep(0.3)` | 壁钟 0.3s |
| 6 | `gz service ... --req 'pause: true'` | 再次暂停（日志条目） |
| 7 | `time.sleep(0.3)` | 0.3s |
| 8 | `wrench/clear` (cylinder) | 清力 |
| 9 | `wrench/persistent` (cylinder, F=...) | 施力 |
| 10 | `time.sleep(0.1)` | 0.1s |
| 11 | `gz service ... --req 'multi_step: 5041'` | 推进 5041 步（约 5.041s 仿真时间） |
| 12 | `time.sleep(5.041)` | 壁钟等 5.041s |
| 13 | `wrench/clear` (cylinder) | 清力 |
| 14 | `multi_step: 1` + 读 topic | 触发发布并读取 → **得到 P1** |
| — | 关闭进程 1 | kill |

**施力前的“起始状态”**：由步骤 3→4 之间仿真实际运行的时间决定。  
从 `pause: false` 发出到 `pause: true` 被处理，中间有一段**壁钟时间 Δt₁**（几十毫秒量级），仿真在 RTF=1 下会推进 **Δt₁ 秒**。因此“施力瞬间”的初始状态 = SDF 初态 + 仿真运行 **Δt₁** 秒后的状态。

---

### Run 2（第二个 Gazebo 进程）

| 序号 | 操作 | 说明 |
|------|------|------|
| 1 | `time.sleep(2)` | 日志中的 “Wait for second Gazebo to start” |
| 2 | `kill_all_gz()` + `time.sleep(1)` | 清理并再等 1s |
| 3 | `Popen("gz sim a.sdf")` | **重新**启动进程 2 |
| 4 | `time.sleep(2)` | 壁钟等 2s |
| 5 | `gz service ... --req 'pause: false'` | **激活：开始仿真** |
| 6 | `gz service ... --req 'pause: true'` | **激活：立即暂停** |
| 7 | `time.sleep(0.3)` | 壁钟 0.3s |
| 8 | 同 Run 1 的 6–13 | pause、清力、施力、multi_step 5041、清力 |
| 9 | `multi_step: 1` + 读 topic | **得到 P2** |
| — | 关闭进程 2 | kill |

**施力前的“起始状态”**：同样由步骤 5→6 之间的壁钟时间 **Δt₂** 决定。  
Run 2 的进程是新建的，调度、负载与 Run 1 不同，因此 **Δt₂ 与 Δt₁ 一般不同**，仿真在“激活”阶段跑过的仿真时间也就不同。

---

## 2. 是否存在实验误差？结论：存在

- **理想情况**：两次运行应满足「相同初始状态 + 相同力 + 相同步数」→ 比较 P1 与 P2 才是在测**仿真器本身的确定性**。
- **当前实现**：
  - “初始状态” = 激活阶段（play → pause）结束时的状态。
  - 激活阶段仿真运行时长 = **Δt**（壁钟时间，由进程调度、负载决定），**未加控制**。
  - Run 1 为 **Δt₁**，Run 2 为 **Δt₂**，**Δt₁ ≠ Δt₂** 是常态。
- **后果**：  
  施力时，Run 1 的 cylinder 已在斜面上运动了 **Δt₁** 秒，Run 2 运动了 **Δt₂** 秒。若 Δt₁ ≠ Δt₂，则**两次的起始位置（和速度）本就不一样**，再各跑 5041 步，P1 与 P2 的差异中会混入：
  - **实验误差**：起始状态不同导致的轨迹差；
  - **可能的真实非确定性**：即使起始状态相同，两进程结果仍不同。

因此：**“两次实验中开始的位置都不一样” 的担心成立**——至少从机制上，Run 2 相对 Run 1 整段流程更晚、且是另一个进程，激活阶段仿真时间与 Run 1 不同，会带来**起始状态不一致**，属于实验误差来源。

---

## 3. 误差量级是否足以解释观察到的 P1–P2 差？

- 激活阶段若差 **0.01–0.05 s** 仿真时间，在斜面上 cylinder 会多滑一点，起始位置差可能在 **厘米到十几厘米** 量级。
- 再经过 5 s 的施力与碰撞，轨迹会进一步分离，**放大到 0.3–1.5 m 是 plausible 的**。
- 因此：复现中看到的 **Run1 vs Run2 位置差约 0.3–1.7 m**，**不能全部归因于“真实非确定性”**，有一部分很可能来自**未控制的起始状态差异**（实验误差）。
- 原始实验 _36 的 **11 m** 差，若当时两进程的激活时间差更大，或叠加真实非确定性，也会被放大。

---

## 4. 建议：如何减少实验误差、更干净地测确定性

1. **控制激活阶段的仿真时间（推荐）**  
   不用「play → pause」这种依赖壁钟的激活，改为：
   - 启动后先 `pause: true`（若默认未暂停），再发一次 `multi_step: 1` 或 `multi_step: 10`，使两轮都在**相同、固定的仿真步数**后进入“施力前”状态；
   - 再执行：清力 → 施力 → multi_step 5041 → 清力 → 读 P。

2. **至少记录并报告“施力前”状态**  
   在施力前、激活之后，对两轮各做一次 `multi_step: 1` + 读 cylinder 位姿（或读 world state），得到：
   - Run 1 施力前位置 **Q1**，Run 2 施力前位置 **Q2**。  
   若 |Q1 − Q2| 已经很大（例如 > 0.01 m），说明当前流程下**起始状态不一致**，此时 P1 与 P2 的差不能单纯解释为“仿真器非确定性”，应在报告中说明。

3. **可选：施力前强制对齐到同一状态**  
   若 Gazebo 支持，可在激活后、施力前用 `set_pose` 等把 cylinder（及需要的话其他物体）设成**同一组初始位姿**，再施力、步进，这样 P1–P2 的差异才更纯粹地反映“同一初态下的非确定性”。

---

## 5. 小结

| 问题 | 结论 |
|------|------|
| 两次实验开始位置是否一样？ | **不一定一样**。由激活阶段（play→pause）的壁钟时间 Δt 决定，Run 1 与 Run 2 的 Δt 未控制，通常不同。 |
| 是否存在实验误差？ | **存在**。起始状态未统一，部分 P1–P2 差可能来自“初态不同”而非“仿真器非确定性”。 |
| 复现看到的 0.3–1.7 m 差 | 可能混合了：① 初态不同导致的轨迹差；② 真实非确定性。 |
| 如何改进 | 用固定步数激活（如 multi_step）替代 play→pause，或在施力前记录/对齐初态并报告。 |

因此，在写 issue 或下“非确定性”结论时，更稳妥的表述是：  
在**当前复现流程**下，两进程的 Run1/Run2 终点存在稳定差异；由于**激活阶段未控制**，部分差异可能来自起始状态不一致（实验误差），建议在**控制初态或报告施力前位姿**后，再对“仿真器本身是否确定性”做更严格判断。

---

## 6. 该误差的影响范围

### 6.1 是否影响所有 Determinism 类型测试？

**是。** 所有 Determinism 测试（无论哪份 SDF、哪个模型）都采用同一流程：进程 1 启动 → play→pause 激活 → 施力+步进 → 读 P1；进程 2 再启动 → play→pause 激活 → 施力+步进 → 读 P2。两轮激活的壁钟时间 Δt₁、Δt₂ 均未控制，因此**所有** Determinism 实验都可能存在“两轮起始状态不完全一致”的实验误差。

### 6.2 对其他类型测试是否有影响？

**影响方式不同，且通常不如 Determinism 严重。**

- **Zero-Input / Force Removal / Force Isolation / Temporal Monotonicity**：都在**单个** Gazebo 进程内完成，只有**一次**启动和一次 play→pause 激活。激活时间只影响这一次的“起始状态”，不会出现“两个进程、两个不同初态”的对比。
- 这些测试的结论多是**同一次运行内的相对关系**（例如：无力时是否漂移、撤力后速度是否跳变、两模型是否隔离、位移是否单调），不依赖“与另一次进程的绝对位置一致”。因此，激活时间波动不会像 Determinism 那样直接造成**两进程初态不同**的系统性偏差。
- 若有人**重复跑**同一 SDF 的同一非-Determinism 测试，两次的绝对位置可能因激活时间略有不同而略有差异，但判据（漂移阈值、速度比、单调性等）仍是单次运行内的量，不受“两进程初态差”影响。

**小结**：  
- **Determinism**：该误差**会影响**，且影响所有此类测试；复现脚本已增加“施力前 Q1/Q2 与初态差”的记录与报告，便于你判断是否混入实验误差。  
- **其他类型**：无“两进程比初态”的问题；仅在“绝对位置随重跑变化”的意义上有轻微影响，一般不影响 PASS/FAIL 的解读。

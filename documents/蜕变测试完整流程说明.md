# 蜕变测试完整流程说明

本文档以**时间比例测试（time_scaling）**为例，详细描述从启动Gazebo到关闭Gazebo的完整测试流程，包括所有需要写入`experiment_log.json`的操作。

## 测试概述

**时间比例测试**验证在不同`real_time_factor`（实时因子）下，相同物理时间内模型的位置是否相同。理论上，`real_time_factor`只影响模拟速度，不应影响物理行为。

## 完整测试流程

### 阶段1：启动和初始化

#### 1.1 启动Gazebo模拟器
- **操作类型**: `command` (launch)
- **命令**: `gz sim <sdf_file> -r --record-path <log_path>`
- **功能**: 启动Gazebo，加载SDF文件，启用自动运行模式(-r)，开始录制模拟过程
- **等待时间**: 5.0秒（等待Gazebo完全启动）
- **记录内容**:
  ```json
  {
    "type": "command",
    "command_type": "launch",
    "command": "gz sim ...",
    "wait_after": 5.0,
    "description": "Launch Gazebo with recording",
    "timestamp": ...
  }
  ```

#### 1.2 等待Gazebo启动
- **操作类型**: `sleep`
- **持续时间**: 5.0秒
- **功能**: 等待Gazebo进程完全启动
- **记录内容**:
  ```json
  {
    "type": "sleep",
    "duration": 5.0,
    "description": "Wait for Gazebo to start",
    "timestamp": ...
  }
  ```

#### 1.3 等待模拟器完全启动
- **操作类型**: `sleep`
- **持续时间**: 2.0秒
- **功能**: 确保模拟器内部状态完全初始化
- **记录内容**:
  ```json
  {
    "type": "sleep",
    "duration": 2.0,
    "description": "Wait for simulation to fully start",
    "timestamp": ...
  }
  ```

#### 1.4 记录测试类型
- **操作类型**: `test_info`
- **功能**: 记录本次执行的蜕变测试类型
- **记录内容**:
  ```json
  {
    "type": "test_info",
    "test_type": "time_scaling",
    "timestamp": ...
  }
  ```

### 阶段2：测试A - 基准测试（rtf=1.0）

#### 2.1 设置物理参数（rtf=1.0）
- **操作类型**: `command` (service)
- **命令**: `gz service -s /world/<world_name>/set_physics ... --req 'max_step_size: 0.001, real_time_factor: 1.0'`
- **功能**: 设置实时因子为1.0（正常速度），作为基准测试
- **等待时间**: 0.2秒（等待参数生效）
- **记录内容**:
  ```json

    "type": "command",
    "command_type": "service",
    "command": "gz service -s ...",
    "timestamp": ...

  {

    "duration": 0.2,
  ```

- **操作类型**: `command` (topic)
- **命令**: `gz topic -t /world/<world_name>/wrench/clear ...`
- **功能**: 清除目标模型（如capsule）上所有已施加的持续力和力矩
- **等待时间**: 0.2秒
- **记录内容**:
  ```json
  {
    "type": "command",
    "command_type": "topic",
    "command": "gz topic -t ...",
    "timestamp": ...
  }
  {
    "type": "sleep",
    "duration": 0.2,
  ```

## 关键改进点

### 获取位置的改进流程

**旧流程**（存在误差）:
1. 施加力
2. 恢复模拟
3. 等待指定时间（模拟在运行）
4. **在模拟运行时获取位置** ← 可能因模拟仍在运行而产生误差

**新流程**（减小误差）:
1. 暂停模拟
2. 施加力
3. 恢复模拟
4. 等待指定时间
5. **暂停模拟** ← 确保模拟完全停止
6. **在暂停状态下获取位置** ← 减小误差
7. 恢复模拟，继续后续操作

## 总结

完整的测试流程包含：
- **启动命令**: 1个（launch）
- **服务命令**: 约15-20个（pause/play/reset/set_pose/set_physics）
- **话题命令**: 约6-8个（施加力/清除力）
- **等待操作**: 约20-25个（各种sleep）
- **信息记录**: 2个（experiment_info, test_info）

所有操作都会按照时间顺序记录到`experiment_log.json`中，可以通过`replay_experiment.py`完整复现。

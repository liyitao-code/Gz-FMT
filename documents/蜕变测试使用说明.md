## 1. 基于速度的蜕变测试

### 1.1 测试原理

**蜕变关系：** 如果模型从初始位置 `(x₀, y₀, z₀)` 开始，以速度 `v` m/s 沿 x 轴正方向运动 `t` 秒，那么最终位置应该是 `(x₀ + v·t, y₀, z₀)` 附近（考虑物理引擎的数值误差）。

**预期行为：** 在理想情况下，匀速运动的位移应该等于速度乘以时间。

### 1.2 代码实现

#### 核心函数：`metamorphic_test_example()`

**位置：** `randomsmith_meta.py` 第 2211-2308 行

**函数签名：**
```python
def metamorphic_test_example(self, velocity_x=None, test_duration=5.0):
    """
    蜕变测试：随机选择一个模型，让它从当前位置沿x轴正方向运动，验证t秒后的位置
    
    Args:
        velocity_x: x轴方向的速度（m/s），如果为None则随机生成
        test_duration: 测试持续时间（秒）
    
    Returns:
        (model_name, initial_position, final_position, expected_position, success) 或 None
    """
```

**测试流程：**

1. **选择模型**：从场景中随机选择一个可用模型（排除边界模型）
2. **获取初始位置**：使用 `get_model_pose_from_topic()` 获取模型的实时位置
3. **生成速度**：如果未指定，随机生成 0.5-2.0 m/s 的速度
4. **施加力**：调用 `func_apply_model_force()` 施加持续力，使模型沿 x 轴运动
   - 力的大小：`force_x = velocity_x * 50.0`（简化处理，假设模型质量）
5. **等待运动**：等待 `test_duration` 秒（真实时间）
6. **获取最终位置**：再次使用 `get_model_pose_from_topic()` 获取最终位置
7. **验证结果**：计算预期位置并与实际位置比较
   - 预期位置：`(initial_x + velocity_x * test_duration, initial_y, initial_z)`
   - 容差：0.5 米（可调整）

#### 辅助函数

**`get_model_pose_from_topic(model_name)`**（第 1278-1348 行）
- 通过订阅 `/world/<world_name>/dynamic_pose/info` 或 `/world/<world_name>/pose/info` topic 获取模型的实时位置
- 返回 `(x, y, z)` 元组

**`func_apply_model_force(...)`**（第 2098-2158 行）
- 对模型施加力或力矩
- 使用 `gz topic` 命令发布 `EntityWrench` 消息到 `/world/<world_name>/wrench/persistent` topic

### 1.4 人工复现步骤

1. 确保 SDF 文件中加载了 `gz-sim-apply-link-wrench-system` 插件（虽然当前代码中已注释掉 `add_required_plugin()` 调用）
2. 启动 Gazebo 模拟器：
   ```bash
   gz sim your_world.sdf -r
   ```
   `-r` 参数确保模拟器自动运行（非暂停状态）

#### 步骤 1：获取模型初始位置

```bash
# 获取 world 名称（假设为 "shapes"）

# 或者只获取动态模型的位置
gz topic -e -t /world/shapes/dynamic_pose/info -m gz.msgs.Pose_V -n 1
```

从输出中找到目标模型的名称和位置，例如：
```
pose {
  name: "ellipsoid"
  position {
    x: 0.0
    y: 0.0
    z: 0.5
  }
  ...
}
```

#### 步骤 2：施加持续力使模型运动

假设：
- world_name = `shapes`
- model_name = `ellipsoid`
- 目标速度：1.0 m/s（沿 x 轴）
- 力的大小：`force_x = 1.0 * 50.0 = 50.0 N`

```bash
gz topic -t /world/shapes/wrench/persistent -m gz.msgs.EntityWrench -p 'entity: {name: "ellipsoid", type: MODEL}, wrench: {force: {x: 50.0, y: 0.0, z: 0.0}, torque: {x: 0.0, y: 0.0, z: 0.0}}'
```

#### 步骤 3：等待指定时间

等待 5 秒（真实时间）：
```bash
sleep 5
```

#### 步骤 4：获取模型最终位置

```bash
gz topic -e -t /world/shapes/dynamic_pose/info -m gz.msgs.Pose_V -n 1
```

#### 步骤 5：验证结果

计算预期位置：
- 初始位置：`(0.0, 0.0, 0.5)`
- 速度：1.0 m/s
- 时间：5.0 秒
- 预期位置：`(0.0 + 1.0 * 5.0, 0.0, 0.5) = (5.0, 0.0, 0.5)`

比较实际位置与预期位置，误差应该在 0.5 米以内。

#### 清除持续力（可选）

```bash

```

## 2. 状态回溯测试

### 2.1 测试原理

**蜕变关系：** 在时间 `t_a` 记录所有模型的状态，继续运行 `t_b` 秒后，回溯到时间 `t_a` 的状态，所有模型的状态应该与记录时完全一致。

**预期行为：** 状态恢复功能应该能够完整地恢复模拟的物理状态（位置、速度、角度等），回溯后的状态应该与记录时相同。

### 2.2 代码实现

#### 核心函数：`metamorphic_test_rewind()`

**位置：** `randomsmith_meta.py` 第 1714-1864 行

**函数签名：**
```python
def metamorphic_test_rewind(self, record_time_a=2.0, run_time_b=3.0):
    """
    蜕变测试：利用 Gazebo 的回溯功能
    
    Args:
        record_time_a: 记录状态的时间点（秒），从模拟开始计算
        run_time_b: 继续运行的时间（秒）
    
    Returns:
        (success, record_time, rewind_time, state_before, state_after, differences) 或 None
    """
```

**测试流程：**

1. **等待到记录时间点**：使用 `get_simulation_time()` 监控模拟时间，等待到 `record_time_a` 秒
2. **保存完整状态**：调用 `save_simulation_state()` 保存完整的模拟状态（包括位置、速度等）
3. **记录模型位置**：调用 `record_all_models_state()` 记录所有模型的位置和角度（用于对比）
4. **继续运行**：等待模拟时间运行 `run_time_b` 秒
5. **恢复状态**：调用 `restore_simulation_state()` 恢复到保存的状态
6. **获取恢复后状态**：再次调用 `record_all_models_state()` 获取恢复后的模型状态
7. **对比状态**：调用 `compare_models_state()` 对比记录的状态和恢复后的状态
   - 位置容差：1e-6 米（默认）
   - 角度容差：1e-6（默认）

#### 辅助函数

**`get_simulation_time()`**（第 1350-1400 行）
- 订阅 `/stats` topic 获取当前模拟时间
- 返回 `(sec, nsec)` 元组

**`save_simulation_state()`**（第 1468-1498 行）
- 调用 `/world/<world_name>/state` 服务获取完整的序列化状态
- 返回 `SerializedState` 对象

**`restore_simulation_state(saved_state)`**（第 1500-1550 行）
- 调用 `/world/<world_name>/control/state` 服务恢复状态
- 使用 `WorldControlState` 消息发送保存的状态

**`record_all_models_state()`**（第 1402-1466 行）
- 订阅 `/world/<world_name>/pose/info` topic 获取所有模型的位置和角度
- 返回字典：`{model_name: {'position': (x, y, z), 'orientation': (w, x, y, z)}}`

**`compare_models_state(state1, state2)`**（第 1652-1711 行）
- 对比两个状态字典，检查位置和角度是否在容差范围内
- 返回 `(bool, list)`：是否相同，差异列表

### 2.4 人工复现步骤

#### 前置条件

启动 Gazebo 模拟器：
```bash
gz sim your_world.sdf -r
```

#### 步骤 1：等待到记录时间点（例如 2.0 秒）

监控模拟时间：
```bash
gz topic -e -t /stats -m gz.msgs.WorldStatistics -n 1
```

等待输出中的 `sim_time` 达到目标时间（例如 2.0 秒）。

#### 步骤 2：保存完整模拟状态

```bash
# 假设 world_name = "shapes"
gz service -s /world/shapes/state --reqtype gz.msgs.Empty --reptype gz.msgs.SerializedStepMap --timeout 3000 --req ''
```

**注意：** 这个命令会返回一个 `SerializedStepMap` 消息，包含完整的模拟状态。由于消息很大，通常需要通过程序处理，而不是手动复制。

#### 步骤 3：记录所有模型的位置和角度

```bash
gz topic -e -t /world/shapes/pose/info -m gz.msgs.Pose_V -n 1
```

记录输出中所有模型的 `position` 和 `orientation` 信息。

#### 步骤 4：继续运行指定时间（例如 3.0 秒）

继续监控模拟时间，等待从 2.0 秒运行到 5.0 秒：
```bash
# 持续监控直到达到目标时间
while true; do
    gz topic -e -t /stats -m gz.msgs.WorldStatistics -n 1 | grep sim_time
    sleep 0.5
done
```

#### 步骤 5：恢复模拟状态

**方法 1：使用 `/control/state` 服务（推荐）**

```bash
# 注意：需要将之前保存的 SerializedState 消息作为请求发送
# 由于消息格式复杂，通常需要通过程序调用
# 这里仅展示服务格式
gz service -s /world/shapes/control/state --reqtype gz.msgs.WorldControlState --reptype gz.msgs.Boolean --timeout 3000 --req '<WorldControlState消息>'
```

**方法 2：使用 Python 脚本（更实际）**

由于状态消息格式复杂，建议使用 Python 脚本：

```python
from gz.transport13 import Node
from gz.msgs11.empty_pb2 import Empty
from gz.msgs11.serialized_map_pb2 import SerializedStepMap
from gz.msgs11.world_control_state_pb2 import WorldControlState
from gz.msgs11.boolean_pb2 import Boolean

node = Node()
world_name = "shapes"

# 1. 保存状态
service_name = f"/world/{world_name}/state"
request = Empty()
result, response = node.request(service_name, request, Empty, SerializedStepMap, 3000)
if result and response.HasField('state'):
    saved_state = response.state
    
    # 2. 等待一段时间（模拟继续运行）
    import time
    time.sleep(3.0)  # 等待3秒
    
    # 3. 恢复状态
    restore_service = f"/world/{world_name}/control/state"
    restore_request = WorldControlState()
    restore_request.state.CopyFrom(saved_state)
    result, response = node.request(restore_service, restore_request, WorldControlState, Boolean, 3000)
    if result and response.data:
        print("State restored successfully")
```

#### 步骤 6：获取恢复后的模型状态

```bash
gz topic -e -t /world/shapes/pose/info -m gz.msgs.Pose_V -n 1
```

#### 步骤 7：对比状态

手动对比步骤 3 和步骤 6 中记录的所有模型的位置和角度，应该完全相同（在数值误差范围内）。

### 3.1 主测试循环

**位置：** `generate_and_test_commands()` 函数（第 850-1003 行）

## 4. 测试结果说明

### 4.1 结果文件

每次测试会在测试目录下生成 `metamorphic_test_result.txt` 文件，包含：

#### 运动测试结果格式

```
Test Type: Motion Test
Model: ellipsoid
Initial Position: (0.000, 0.000, 0.500)
Final Position: (5.123, 0.000, 0.500)
Expected Position: (5.000, 0.000, 0.500)
Test Duration: 5.00 s
Result: PASSED
Error: x=0.123, y=0.000, z=0.000
```

#### 回溯测试结果格式

```
Test Type: Rewind Test
Record Time: 2.00 s
Run Time: 3.00 s
Result: PASSED
Number of Models: 5
Differences: []
```

如果测试失败，会显示差异信息。

### 4.2 失败标记

如果测试失败，会在测试目录下创建标记文件：
- `METAMORPHIC_TEST_FAILED`：仅蜕变测试失败
- `BOTH_CRASH_AND_METAMORPHIC_FAIL`：同时发生崩溃和测试失败

### 4.3 容差设置

- **运动测试**：位置误差容差为 0.5 米（可在代码中调整）
- **回溯测试**：位置容差为 1e-6 米，角度容差为 1e-6（可在 `compare_models_state()` 中调整）

## 5. 常见问题

### 5.1 模型不移动

**可能原因：**
- 模拟器处于暂停状态（应使用 `-r` 参数启动）
- 力的大小不足（需要根据模型质量调整）
- 模型被固定或碰撞到其他物体

**解决方法：**
- 确保使用 `gz sim ... -r` 启动
- 增加力的大小（调整 `force_x = velocity_x * 50.0` 中的系数）
- 检查模型是否与其他物体碰撞

### 5.2 状态恢复失败

**可能原因：**
- `/world/<world_name>/control/state` 服务不可用
- 状态消息格式错误
- 模拟器版本不兼容

**解决方法：**
- 检查服务是否存在：`gz service --list | grep control/state`
- 确保使用兼容的 Gazebo 版本
- 查看调试输出中的错误信息

### 5.3 获取位置失败

**可能原因：**
- Topic 不存在或未订阅成功
- 模型名称错误
- 模拟器未完全启动

**解决方法：**
- 检查 topic 是否存在：`gz topic --list | grep pose/info`
- 确认模型名称正确
- 增加等待时间确保模拟器完全启动

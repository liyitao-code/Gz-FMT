# 匀速运动实现原理说明

本文档解释当前测试代码中如何实现模型的匀速运动，以及相关的物理原理。

---

## 1. 当前实现方式

### 1.1 使用持续力（Persistent Force）

**代码位置：** `randomsmith_meta.py` 第 2260-2266 行

```python
force_cmd = self.func_apply_model_force(
    model_name=model_name,
    force_x=force_x,
    force_y=0.0,
    force_z=0.0,
    persistent=True  # 关键：使用持续力
)
```

**关键点：**
- `persistent=True` 表示使用**持续力**，而不是瞬时力
- 持续力会在**每个物理时间步**都应用，直到被清除

### 1.2 持续力 vs 瞬时力

#### 持续力（Persistent Wrench）

**Topic：** `/world/<world_name>/wrench/persistent`

**行为：**
- 力会在每个物理时间步都应用
- 直到通过 `/world/<world_name>/wrench/clear` topic 清除
- 适合需要持续推动模型的场景

**Gazebo 源码实现：** `src/gz-sim/src/systems/apply_link_wrench/ApplyLinkWrench.cc` 第 298-302 行
```cpp
// Apply persistent wrenches at every time step
for (auto msg : this->dataPtr->persistentWrenches)
{
    // 每个时间步都应用这个力
    link.AddWorldWrench(_ecm, force, torque, offset);
}
```

#### 瞬时力（Instantaneous Wrench）

**Topic：** `/world/<world_name>/wrench`

**行为：**
- 力只在一个物理时间步应用
- 应用后立即移除
- 适合需要瞬间冲击的场景

**Gazebo 源码实现：** `src/gz-sim/src/systems/apply_link_wrench/ApplyLinkWrench.cc` 第 269-296 行
```cpp
// Apply instantaneous wrenches
while (!this->dataPtr->newWrenches.empty())
{
    auto msg = this->dataPtr->newWrenches.front();
    // 只应用一次，然后从队列中移除
    link.AddWorldWrench(_ecm, force, torque, offset);
    this->dataPtr->newWrenches.pop();
}
```

## 2. Gazebo 中的摩擦

### 2.1 摩擦参数

**Gazebo 中确实有摩擦！** 默认摩擦参数如下：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `static_friction` (μs) | 0.9 | 静摩擦系数 |
| `dynamic_friction` (μd) | 0.9 | 动摩擦系数 |
| `viscous_friction` (μv) | 0.0 | 粘性摩擦系数 |

**位置：** SDF 文件中的 `<surface><friction>` 元素，或物理引擎的全局设置。

### 2.2 摩擦对运动的影响

#### 情况 1：模型在空气中（无接触，无摩擦）

- **持续力会导致持续加速**
- 根据牛顿第二定律：`F = ma`，加速度 `a = F/m`
- 速度会随时间线性增加：`v(t) = v₀ + a·t`
- **不是匀速运动！**

#### 情况 2：模型在地面上（有接触，有摩擦）

- **摩擦力：** `F_friction = μ · N`（其中 N 是法向力，通常等于重力 `mg`）
- **如果驱动力 > 摩擦力：** 模型会加速，直到达到某个速度
- **如果驱动力 = 摩擦力：** 模型会以**匀速**运动（理想情况）
- **如果驱动力 < 摩擦力：** 模型可能静止或减速

## 3. 当前代码的问题

### 3.1 力的计算方式

**代码位置：** `randomsmith_meta.py` 第 2258 行

```python
force_x = velocity_x * 50.0  # 根据模型质量调整，这里使用较大的系数
```

**问题：**
1. **这个计算方式不正确**，它假设力与速度成正比，但物理上：
   - 力产生加速度，而不是直接产生速度
   - 要达到匀速，需要力等于摩擦力（如果有摩擦）

2. **没有考虑摩擦：**
   - 如果模型在空气中，持续力会导致加速，不是匀速
   - 如果模型在地面上，需要知道摩擦系数才能计算正确的力

### 3.2 测试假设

**当前测试的假设：**
- 模型会以接近匀速运动
- 位移 = 速度 × 时间

**实际情况：**
- 如果模型在空气中：模型会加速，实际位移 > 预期位移
- 如果模型在地面上：取决于力与摩擦的关系

## 4. 正确的匀速运动实现方式

### 4.1 方法 1：使用速度控制（推荐）

**如果模型支持速度控制，直接设置速度：**

```python
# 使用 VelocityControl 系统（如果模型有 free_group）
# 这需要模型没有关节约束
```

**限制：** 不是所有模型都支持速度控制。

### 4.2 方法 2：计算平衡力（有摩擦时）

**如果模型在地面上，需要计算平衡力：**

```python
# 假设模型质量为 m，摩擦系数为 μ
# 重力：G = m * g
# 摩擦力：F_friction = μ * G = μ * m * g
# 要达到匀速，需要：F_drive = F_friction

mass = 1.0  # 需要获取模型的实际质量
mu = 0.9    # 摩擦系数（默认值）
g = 9.8     # 重力加速度

# 计算平衡力
friction_force = mu * mass * g
force_x = friction_force  # 如果沿 x 轴运动，且重力在 z 轴
```

**问题：** 需要知道模型质量和摩擦系数。

### 4.3 方法 3：使用阻尼（无摩擦时）

**如果模型在空气中，可以添加线性阻尼来达到匀速：**

```python
# 在 SDF 中添加阻尼参数
# <link>
#   <linear_velocity_damping>0.5</linear_velocity_damping>
# </link>

# 然后施加持续力，当力等于阻尼力时，达到匀速
# F_drive = damping * velocity
# 所以：force_x = damping * target_velocity
```

### 4.4 方法 4：分阶段控制

**先加速到目标速度，然后保持：**

```python
# 1. 施加较大的力，加速到目标速度
# 2. 监控速度，当达到目标速度时，调整力的大小
# 3. 保持力等于摩擦力（或阻尼力），维持匀速
```

## 5. 当前测试的实际行为

### 5.1 如果模型在空气中

**物理过程：**
1. 施加持续力 `F`
2. 模型加速：`a = F/m`
3. 速度增加：`v(t) = a·t`
4. 位移：`s(t) = ½·a·t²`（不是 `v·t`！）

**测试结果：**
- 实际位移 > 预期位移（因为加速）
- 测试可能失败，因为假设是匀速运动

### 5.2 如果模型在地面上

**物理过程：**
1. 施加持续力 `F`
2. 如果 `F > F_friction`：模型加速
3. 如果 `F = F_friction`：模型匀速（理想情况）
4. 如果 `F < F_friction`：模型可能静止

**测试结果：**
- 取决于力与摩擦的关系
- 如果力太大，会加速；如果力太小，可能不动

## 6. 改进建议

### 6.1 短期改进

1. **增加容差：** 当前容差是 0.5 米，可以适当增加，因为实际是加速运动
2. **修改预期：** 如果知道模型在空气中，使用 `s = ½·a·t²` 计算预期位移
3. **记录实际速度：** 在测试中记录模型的实际速度，验证是否符合预期

### 6.2 长期改进

1. **获取模型质量：** 从 SDF 或场景信息中获取模型的实际质量
2. **检测摩擦：** 检测模型是否与地面接触，获取摩擦系数
3. **智能计算力：** 根据质量和摩擦计算正确的力
4. **使用速度控制：** 如果模型支持，直接设置速度而不是施加力


## 7. 总结

### 当前实现：
- ✅ 使用**持续力**（`persistent=True`）
- ✅ Gazebo 中**有摩擦**（默认 μ = 0.9）
- ❌ 力的计算方式**不正确**（假设力与速度成正比）
- ❌ 没有考虑摩擦和加速的影响

### 实际行为：
- 如果模型在空气中：**加速运动**，不是匀速
- 如果模型在地面上：取决于力与摩擦的关系

### 建议：
- 短期：增加容差，或修改预期计算方式
- 长期：获取模型质量和摩擦信息，正确计算力

# 已实现的蜕变关系总结

根据代码和文档分析，当前系统已实现 **6种蜕变测试关系**，其中5种为主要测试关系，1种为辅助测试关系。

---

## 主要蜕变测试关系

### 1. 基于速度的蜕变测试（Motion Test）

**函数名**: `metamorphic_test_example()`  
**测试类型标识**: `'motion'`  
**代码位置**: `randomsmith_meta.py` 第 2673-2762 行

**蜕变关系描述**:
- 如果模型从初始位置 `(x₀, y₀, z₀)` 开始，以速度 `v` m/s 沿 x 轴正方向运动 `t` 秒
- 那么最终位置应该是 `(x₀ + v·t, y₀, z₀)` 附近（考虑物理引擎的数值误差）

**测试流程**:
1. 随机选择一个模型（当前固定为 "ellipsoid"）
2. 获取初始位置（通过 topic 订阅）
3. 暂停模拟
4. 施加持续力使模型沿 x 轴运动（力的大小：`force_x = velocity_x * 50.0`）
5. 恢复模拟，开始计时
6. 等待指定时间（`test_duration` 秒，默认 3-8 秒随机）
7. 暂停模拟
8. 获取最终位置（在暂停状态下，减小误差）
9. 恢复模拟
10. 计算预期位置并与实际位置比较（容差：0.5 米）

**返回值**: `(model_name, initial_pos, final_pos, expected_pos, success)`

**文档说明**: 
- `documents/蜕变测试使用说明.md` 第 1 节详细说明
- `documents/METAMORPHIC_RELATIONS.md` 第 1 节

### 2. 状态回溯测试（Rewind Test）

**函数名**: `metamorphic_test_rewind()`  
**测试类型标识**: `'rewind'`  
**代码位置**: `randomsmith_meta.py` 第 1985-2135 行

**蜕变关系描述**:
- 在时间 `t_a` 记录所有模型的状态（位置、角度等）
- 继续运行 `t_b` 秒
- 回溯到时间 `t_a` 时的状态
- 回溯后的状态应该与记录时的状态完全一致

**测试流程**:
1. 获取初始模拟时间
2. 等待模拟时间达到 `record_time_a` 秒
3. 记录所有模型的完整状态（使用 `save_simulation_state()` 和 `record_all_models_state()`）
4. 继续运行 `run_time_b` 秒（基于模拟时间）
5. 使用保存的状态恢复模拟（`restore_simulation_state()`）
6. 等待状态恢复完成
7. 获取回溯后的所有模型状态
8. 比较回溯前后的状态，找出差异

**参数**:
- `record_time_a`: 记录状态的时间点（默认 2.0 秒）
- `run_time_b`: 继续运行的时间（默认 3.0 秒）

**返回值**: `(success, record_time, rewind_time, state_before, state_after, differences)`

**文档说明**: 
- `documents/蜕变测试使用说明.md` 第 2 节详细说明
- `documents/METAMORPHIC_RELATIONS.md` 第 2 节

### 3. 力的可加性测试（Force Additivity Test）

**函数名**: `metamorphic_test_force_additivity()`  
**测试类型标识**: `'force_additivity'`  
**代码位置**: `randomsmith_meta.py` 第 2808-3019 行

**蜕变关系描述**:
- 对同一模型同时施加多个力 F1, F2, ..., Fn
- 结果应该等于这些力的矢量和 F_total = F1 + F2 + ... + Fn
- 即：同时施加 F1 和 F2 的效果 = 施加 F_total 的效果

**测试流程**:
1. 随机选择一个模型，获取初始位置和角度
2. **测试A**：
   - 清除之前的力
   - 暂停模拟
   - 使用 `gz topic` 命令同时施加力 F1 和 F2（通过快速连续发布两个 EntityWrench 消息）
   - 恢复模拟，开始计时
   - 等待 `test_duration` 秒
   - 暂停模拟
   - 获取最终位置 P1（在暂停状态下）
   - 恢复模拟
3. **测试B**：
   - 清除力
   - 重置模拟到初始状态（`reset_simulation()`）
   - 恢复模型到初始位置和角度（`set_model_pose()`）
   - 暂停模拟
   - 施加力 F_total = F1 + F2（矢量相加）
   - 恢复模拟，开始计时
   - 等待 `test_duration` 秒（与测试A相同）
   - 暂停模拟
   - 获取最终位置 P2（在暂停状态下）
   - 恢复模拟
4. 验证 P1 ≈ P2（容差：0.5 米）

**关键实现细节**:
- 使用暂停-施加力-恢复的流程确保力同时施加
- 两次测试使用完全相同的力值（保存为实例变量）
- 从恢复模拟开始计时，确保时间一致性

**返回值**: `(model_name, initial_pos, pos_with_f1_f2, pos_with_f_total, success, error_info)`

**文档说明**: 
- `documents/METAMORPHIC_RELATIONS.md` 第 3 节（建议实现，现已实现）

### 4. 时间比例测试（Time Scaling Test）

**函数名**: `metamorphic_test_time_scaling()`  
**测试类型标识**: `'time_scaling'`  
**代码位置**: `randomsmith_meta.py` 第 3269-3562 行

**蜕变关系描述**:
- 通过调整 `real_time_factor` 参数，调节仿真速度比例
- 物理行为应该有比例关系或性质相同
- 如果 `real_time_factor = r`，那么在相同物理时间内，模型的位置应该相同
- 即：`real_time_factor` 只影响模拟速度，不影响物理行为

**测试流程**:
1. 随机选择一个模型，获取初始位置和角度
2. 随机生成一个力 F1（保存为实例变量，确保两次测试使用相同的力）
3. **测试A**（rtf=1.0）：
   - 设置 `real_time_factor = 1.0`
   - 清除力
   - 暂停模拟
   - 施加力 F1
   - 恢复模拟，开始计时
   - 等待 `test_duration` 秒（物理时间）
   - 暂停模拟
   - 获取最终位置 P1（在暂停状态下）
   - 恢复模拟
4. **准备测试B**：
   - 清除力
   - 暂停模拟
   - 恢复模型到初始位置和角度
   - 恢复模拟
   - 重置模拟（`reset_simulation()`）以清除所有速度
   - 设置新的 `real_time_factor`（随机选择 0.5-2.0 之间）
   - 再次恢复模型到初始位置和角度
5. **测试B**（rtf=新值）：
   - 清除力
   - 暂停模拟
   - 施加力 F1（与测试A完全相同）
   - 恢复模拟，开始计时
   - 等待 `test_duration` 秒（物理时间，与测试A相同）
   - 暂停模拟
   - 获取最终位置 P2（在暂停状态下）
   - 恢复模拟
6. 验证 P1 ≈ P2（容差：0.5 米）

**关键实现细节**:
- 两次测试使用完全相同的力值（保存为实例变量 `self._test_force_x/y/z`）
- 使用 `reset_simulation()` 确保模型状态完全重置（包括速度）
- 从恢复模拟开始计时，确保时间一致性

**返回值**: `(model_name, initial_pos, pos_with_rtf1, pos_with_rtf2, rtf1, rtf2, success, error_info)`

**文档说明**: 
- `documents/蜕变测试完整流程说明.md` 详细描述了完整流程
- `documents/METAMORPHIC_RELATIONS.md` 第 5 节（建议实现，现已实现）

### 5. 质量系数测试（Mass Scaling Test）

**函数名**: `metamorphic_test_mass_scaling()`  
**测试类型标识**: `'mass_scaling'`  
**代码位置**: `randomsmith_meta.py` 第 3588-3950 行

**蜕变关系描述**:
- 根据 F=ma，在相同力 F 下，质量 m1 时位移 d1，质量 m2 = k*m1 时位移 d2
- 理论上应该有：d2 = d1/k，即 d1/d2 = k
- 验证质量系数与位移的反比关系

**测试流程**:
1. 随机选择一个模型，获取初始位置和初始质量 m
2. **测试A**（原始质量 m）：
   - 清除之前的力
   - 暂停模拟
   - 施加力 F1（随机生成，保存为实例变量）
   - 恢复模拟，开始计时
   - 等待 `test_duration` 秒
   - 暂停模拟
   - 获取最终位置，计算位移 d1
   - 恢复模拟
3. **准备测试B**：
   - 清除力
   - 重置模型到初始位置
   - 重置模拟以清除速度
   - 修改质量系数为 km（k 是比例因子，范围 0.5-2.0）
     - 通过 `dump_sdf()` 获取模型 SDF
     - 解析并修改所有 link 的质量和惯性矩阵
     - 删除旧模型
     - 使用修改后的 SDF 重新创建模型
   - 再次恢复模型到初始位置
4. **测试B**（新质量 km）：
   - 清除力
   - 暂停模拟
   - 施加力 F1（与测试A完全相同）
   - 恢复模拟，开始计时
   - 等待 `test_duration` 秒（与测试A相同）
   - 暂停模拟
   - 获取最终位置，计算位移 d2
   - 恢复模拟
5. 验证 d1/d2 ≈ k（容差：20%）

**关键实现细节**:
- 使用 `get_model_mass()` 从 scene 中获取模型总质量（所有 link 质量之和）
- 使用 `modify_model_mass()` 修改模型质量：
  - 通过 `dump_sdf()` 获取当前世界 SDF
  - 解析 XML，找到目标模型的所有 link
  - 按比例缩放每个 link 的质量和惯性矩阵
  - 删除旧模型，使用修改后的 SDF 重新创建
- 两次测试使用完全相同的力值（保存为实例变量）
- 从恢复模拟开始计时，确保时间一致性

**返回值**: `(model_name, initial_pos, initial_mass, mass_scale_factor, pos_with_m1, pos_with_m2, d1, d2, success, error_info)`

**辅助函数**:
- `get_model_mass(model_name)`: 获取模型总质量
- `modify_model_mass(model_name, new_mass, mass_scale_factor)`: 修改模型质量

## 辅助蜕变测试关系

### 5. 基于Playback的回溯测试（Playback Rewind Test）

**函数名**: `metamorphic_test_playback_rewind()`  
**测试类型标识**: 无独立标识（作为辅助测试）  
**代码位置**: `randomsmith_meta.py` 第 2141-2278 行

**蜕变关系描述**:
- 使用录像文件（log）在 playback 模式下进行回溯测试
- 在 playback 模式下，使用 seek 功能回溯到指定时间点
- 记录回溯后的模型状态，与原始测试中记录的状态进行对比

**测试流程**:
1. 检查录像文件是否存在
2. 启动 playback 模式的 Gazebo（`gz sim -r --playback <log_path>`）
3. 等待 playback 模式启动（5秒 + 2秒）
4. 根据原始测试类型选择回溯时间点：
   - 对于 motion 测试：回溯到测试开始后 1 秒
   - 对于 rewind 测试：使用原始测试的记录时间点
5. 使用 `seek_in_playback_mode()` 回溯到指定时间
6. 等待 seek 完成（1秒）
7. 记录回溯后的所有模型状态
8. 与原始测试结果对比（如果提供了原始测试结果）
9. 终止 playback 进程

**特点**:
- **常驻测试**：每轮蜕变测试完成后，如果 `enable_playback=True`，会自动执行
- 使用 `--record-path` 参数录制的 log 文件作为输入
- 可以验证 playback 功能的正确性

**返回值**: `(success, details_string)`

**文档说明**: 
- `documents/蜕变测试使用说明.md` 第 2.3 节（Playback 模式回溯）

## 新增蜕变测试关系（P0 优先级）

### 6. 确定性重复测试（Determinism Test）

**函数名**: `_determinism_single_run()` (辅助) + `generate_and_test_commands()` 中的编排逻辑  
**测试类型标识**: `'determinism'`

**蜕变关系描述**:
- 在两个**完全独立的 Gazebo 实例**中执行完全相同的实验
- 相同 SDF、相同模型、相同力、相同步数 → 结果必须完全一致
- **不依赖 reset**，因此能检测与 reset 无关的 bug

**测试流程**:
1. **Gazebo 实例 1**：启动 → 选择模型 → 暂停 → 施力 F → `step_simulation(N)` → 记录位姿 P1 → 关闭
2. 彻底清理所有 Gazebo 进程
3. **Gazebo 实例 2**：启动相同 SDF → 选择相同模型 → 暂停 → 施相同力 F → `step_simulation(N)` → 记录位姿 P2
4. 验证 P1 ≈ P2（容差：0.001 m，即 1mm）

**能检测的 Bug 类型**:
- 竞态条件（多线程插件中的数据竞争）
- 未初始化变量
- 遍历顺序依赖（`unordered_map` 等）
- 随机数种子未固定
- 浮点运算顺序不一致

**特点**:
- 唯一不依赖 `reset_simulation()` 的测试
- 使用两个独立进程，任何非确定性行为都会暴露
- 容差极小（1mm），因为确定性仿真结果理论上完全一致
- Playback 回溯测试在此模式下自动跳过（第一个 Gazebo 的录像可能不完整）

### 7. 对称性/方向不变性测试（Symmetry Test）

**函数名**: `metamorphic_test_symmetry()`  
**测试类型标识**: `'symmetry'`

**蜕变关系描述**:
- 对任意模型，施加相同大小但方向分别为 +x 和 +y 的力
- 对应方向的位移大小应相同（物理空间的各向同性 isotropy）
- 即：Force=(F,0,0) → |dx| ≈ Force=(0,F,0) → |dy|

**测试流程**:
1. 选择模型 M，获取初始位姿
2. 计算力的大小 F_mag（与模型质量成比例）
3. **Test A**：暂停 → 施力 (F_mag, 0, 0) → `step_simulation(N)` → 记录 x 方向位移 dx
4. Reset → 恢复初始状态
5. **Test B**：暂停 → 施力 (0, F_mag, 0) → `step_simulation(N)` → 记录 y 方向位移 dy
6. 验证 |dx| ≈ |dy|（容差：绝对 < 0.5m 或 相对 < 20%）

**附加检查**:
- z 方向位移差异（重力效果应一致）
- 交叉轴位移（x-force 时 y 方向偏移，y-force 时 x 方向偏移）

**能检测的 Bug 类型**:
- 插件硬编码了特定轴方向（如风只沿 x 轴生效）
- 碰撞检测在某些方向上有偏差
- 摩擦力在不同方向上不对称
- 坐标系变换错误
- 惯性矩阵问题

## 新范式蜕变测试关系（v2 新增）

### 8. 零输入稳定性测试（Zero-Input Stability Test）

**函数名**: `metamorphic_test_zero_input_stability()`
**测试类型标识**: `'zero_input_stability'`
**范式**: B — Single-Run Invariant（单次运行不变量检查）

**蜕变关系描述**:
- 不施加任何外力，仅推进仿真
- 模型在水平方向（x, y）不应产生显著位移
- 任何水平漂移都表明存在幽灵力、能量注入或初始化 bug

**测试流程**:
1. 选择模型 M，记录初始位置 P0
2. 暂停仿真，**不施加任何力**
3. `step_simulation(N)`（推进 3-6 秒）
4. 获取位置 P1
5. 验证：|P1.x - P0.x| < 0.05m AND |P1.y - P0.y| < 0.05m
   （允许 z 轴变化，因为重力可能导致模型下沉/稳定）

**能检测的 Bug 类型**:
- 插件幽灵力（风力/浮力插件在不该生效时产生力）
- NaN/inf 传播导致随机移动
- 模型创建时的初始化速度残留
- 求解器能量注入（接触求解器在稳态时注入能量）

**关键区别**:
- **不施加任何力** — 这是所有测试中唯一不需要力的
- **不使用 reset** — 纯粹的单次运行不变量检查
- 施力测试会掩盖小幽灵力（几百 N 的测试力完全掩盖几 N 的幽灵力）

**返回值**: `(model_name, initial_pos, final_pos, drift_x, drift_y, drift_z, success, error_info)`

### 9. 多模型力隔离测试（Force Isolation Test）

**函数名**: `metamorphic_test_force_isolation()`
**测试类型标识**: `'force_isolation'`
**范式**: C — Same-Run Multi-Model（同一运行中比较不同模型）

**蜕变关系描述**:
- 仅对模型 A 施力，远处无接触的模型 B 不应受影响
- 如果 B 产生了水平位移，说明存在力泄漏或目标路由 bug

**测试流程**:
1. 获取所有可测试模型，选择距离 > 2m 的两个模型
2. 记录初始位置：P_target_0, P_bystander_0
3. 仅对 M_target 施加持续力 F
4. `step_simulation(N)`
5. 获取最终位置
6. 检查：M_target 有显著位移（sanity check）
7. 检查：M_bystander 水平漂移 < 0.05m

**能检测的 Bug 类型**:
- EntityWrench 实体名/ID 解析错误
- apply-link-wrench 系统全局力泄漏
- 插件间干扰（一个模型的插件影响另一个）
- 模型实例之间共享不该共享的状态

**关键区别**:
- **不使用 reset** — 在同一运行中同时观察两个模型
- **测试力的隔离性** — 所有其他测试只测一个模型的力学行为
- 需要世界中有至少 2 个距离足够远的可测试模型

**返回值**: `(target_name, bystander_name, target_initial, bystander_initial, target_final, bystander_final, target_displacement, bystander_drift, force_magnitude, success, error_info)`

### 10. 撤力响应测试（Force Removal Response Test）

**函数名**: `metamorphic_test_force_removal()`
**测试类型标识**: `'force_removal'`
**范式**: D — Sequential No-Reset（同一运行分阶段，无 reset）

**蜕变关系描述**:
- 施加持续力后使用 `clear_model_wrench()` 撤除
- 撤力后，模型不应继续加速（应减速或匀速）
- 如果滑行阶段仍在加速，说明力没有被正确清除

**测试流程**:
1. 选择模型 M，记录初始位置 P0
2. 施加持续力 F（+x 方向）
3. `step_simulation(N1)` — 施力阶段（1.5-3 秒）→ 记录 P1
4. `clear_model_wrench(M)` — 撤除力
5. `step_simulation(N2/2)` — 滑行前半段 → 记录 P2
6. `step_simulation(N2/2)` — 滑行后半段 → 记录 P3
7. 估算速度：v_force, v_coast1, v_coast2
8. 检查：滑行阶段加速度 < 1.0 m/s²（不应持续加速）

**能检测的 Bug 类型**:
- `clear_model_wrench` 未正确生效
- 持续力在 clear 后仍然残留
- 插件累积状态未被清理
- wrench 系统记账错误

**关键区别**:
- **不使用 reset** — 两个阶段在同一运行中连续执行
- **测试力的生命周期管理** — 唯一测试 apply → clear → verify 的流程
- 所有其他测试都不检查 `clear_model_wrench()` 是否真正生效

**返回值**: `(model_name, initial_pos, pos_after_force, pos_coast_mid, pos_coast_end, v_force, v_coast1, v_coast2, force_magnitude, success, error_info)`

---

### 11. 时序单调性测试（Temporal Monotonicity Test）

**函数名**: `metamorphic_test_temporal_monotonicity()`
**测试类型标识**: `'temporal_monotonicity'`
**范式**: B — Single-Run Invariant（检查轨迹形状的物理不变量）

**蜕变关系描述**:
- 施加恒定力，在多个时间点采样位移
- 力方向上的位移必须随时间单调递增
- 相邻采样点的位移增量应平滑（无突变/传送）

**测试流程**:
1. 选择模型 M，记录初始位置 P0
2. 施加持续力 F（+x 方向）
3. 循环 K 次（K=8/10/12 随机）：
   - `step_simulation(N/K)` — 每次推进约 0.3-0.75 秒
   - 获取位置 P_i
4. 计算 x 方向位移序列：d_1, d_2, ..., d_K
5. 检查单调性：d_1 < d_2 < ... < d_K
6. 检查平滑性：相邻增量比 < 3.0（无突变/传送）

**能检测的 Bug 类型**:
- 插件偶尔跳过一步不施力（间歇性失效）
- 求解器发散导致位置突变
- 碰撞响应将模型弹回（传送 bug）
- 时间步进累积误差
- 力方向在某些条件下翻转

**关键区别**:
- **唯一检查轨迹形状的测试** — 所有其他测试只比较两个端点
- **不使用 reset** — 单次连续运行中多点采样
- 间歇性 bug（偶尔出错几步然后恢复）会通过所有端点测试但无法通过本测试

**返回值**: `(model_name, initial_pos, trajectory, force_magnitude, monotonic, smooth, violations, success, error_info)`

## 关键改进

### 确定性仿真

所有测试都使用 `step_simulation(num_steps)` 替代 `time.sleep(duration)`：

1. **暂停模拟** → 2. **施加力** → 3. **`step_simulation(N)`**（精确推进 N 步） → 4. **获取位置**（仿真已自动暂停）

这消除了 wall-clock timing 导致的不确定性，确保仿真时间精确可控。

### v2 范式革新

v1 版本的所有测试共享 "Reset-Compare" 范式，只能检测 reset 相关 bug。
v2 引入了 4 种新的结构范式（B/C/D），每种范式针对一个完全不同的 bug 类别，
显著扩展了故障检测覆盖面。

当前激活的 5 种测试（Determinism + 4 种新范式）**全部不依赖 reset**。

# 实验复现说明

## 概述

本系统提供了实验流程记录和自动复现功能。每轮测试会自动生成一个 `experiment_log.json` 文件，记录所有执行的命令和等待时间，可以通过 `replay_experiment.py` 脚本自动复现实验。

## 实验日志格式

`experiment_log.json` 文件是一个 JSON 数组，包含以下类型的条目：

### 1. 实验信息 (experiment_info)
记录实验的基本信息：
```json
{
  "type": "experiment_info",
  "directory": "/path/to/test/directory",
  "sdf_file": "a.sdf",
  "start_time": 1234567890.123,
  "end_time": 1234568000.456,
  "duration": 110.333,
  "test_passed": true,
  "test_type": "time_scaling",
  "timestamp": 1234567890.123
}
```

### 2. 命令 (command)
记录执行的命令：
```json
{
  "type": "command",
  "command_type": "service",  // 或 "topic", "launch"
  "command": "gz service -s /world/shapes/control --reqtype gz.msgs.WorldControl --reptype gz.msgs.Boolean --timeout 10000 --req 'pause: true'",
  "wait_after": 0.3,
  "description": "Pause simulation",
  "timestamp": 1234567890.123
}
```

### 3. 等待 (sleep)
记录等待操作：
```json
{
  "type": "sleep",
  "duration": 5.0,
  "description": "Wait for Gazebo to start",
  "timestamp": 1234567890.123
}
```

### 4. 测试信息 (test_info)
记录测试类型：
```json
{
  "type": "test_info",
  "test_type": "time_scaling",
  "timestamp": 1234567890.123
}
```

## 使用方法

### 1. 自动记录

在运行 `randomsmith_meta.py` 进行测试时，系统会自动在每轮测试的目录下生成 `experiment_log.json` 文件。

例如，如果测试目录是 `/home/liyitao/workspace/meta/test_5/_0/`，则日志文件会保存在：
```
/home/liyitao/workspace/meta/test_5/_0/experiment_log.json
```

### 2. 复现实验

使用 `replay_experiment.py` 脚本可以自动复现实验：

#### 方法1：直接指定日志文件
```bash
python replay_experiment.py /path/to/experiment_log.json
```

#### 方法2：指定测试目录（会自动查找 experiment_log.json）
```bash
python replay_experiment.py /path/to/test/directory
```

#### 示例
```bash
# 方法1
python replay_experiment.py /home/liyitao/workspace/meta/test_5/_0/experiment_log.json

# 方法2
python replay_experiment.py /home/liyitao/workspace/meta/test_5/_0/
```

### 3. 复现过程

复现脚本会：
1. 读取 `experiment_log.json` 文件
2. 按照记录的顺序执行所有命令
3. 在每次命令后等待指定的时间
4. 显示执行进度和结果

### 4. 停止复现

在复现过程中，可以按 `Ctrl+C` 停止。如果 Gazebo 进程正在运行，脚本会自动终止它。

## 注意事项

1. **路径问题**：复现时，确保 SDF 文件路径正确。如果测试目录被移动，可能需要手动修改日志文件中的路径。

2. **环境依赖**：复现需要相同的 Gazebo 环境，包括：
   - 相同的 Gazebo 版本
   - 相同的模型文件
   - 相同的系统配置

3. **时间精度**：复现时会尽量按照原始时间间隔执行，但由于系统负载等因素，实际时间可能会有微小差异。

4. **随机性**：如果原始测试包含随机元素（如随机选择的模型、随机生成的力等），复现时这些值会从日志中读取，确保一致性。

## 日志文件位置

实验日志文件保存在每轮测试的目录下：
```
<test_directory>/experiment_log.json
```

例如：
```
/home/liyitao/workspace/meta/test_5/_0/experiment_log.json
/home/liyitao/workspace/meta/test_5/_1/experiment_log.json
...
```

## 故障排除

### 问题1：日志文件不存在
**原因**：测试可能没有正常完成，或者日志记录功能未启用。

**解决**：检查测试是否正常完成，查看是否有错误信息。

### 问题2：复现时命令执行失败
**原因**：可能是路径问题、环境问题或 Gazebo 未正确安装。

**解决**：
- 检查命令中的路径是否正确
- 确认 Gazebo 环境是否正确配置
- 查看错误信息，定位具体问题

### 问题3：复现结果与原始测试不一致
**原因**：可能是环境差异、时间精度问题或随机性导致的。

**解决**：
- 确保使用相同的 Gazebo 版本和配置
- 检查系统负载是否影响时间精度
- 查看日志文件，确认所有步骤都被正确记录

## 扩展功能

如果需要扩展记录功能，可以：

1. **添加自定义记录类型**：在 `randomsmith_meta.py` 中添加新的记录类型
2. **修改复现逻辑**：在 `replay_experiment.py` 中添加对新类型的处理
3. **添加验证功能**：在复现过程中添加结果验证

## 相关文件

- `randomsmith_meta.py`：主测试脚本，包含实验记录功能
- `replay_experiment.py`：实验复现脚本
- `experiment_log.json`：实验日志文件（每轮测试自动生成）

# Gazebo 蜕变测试关系设计文档

本文档列出了可以在 Gazebo 模拟器中实现的蜕变测试关系（Metamorphic Relations）。

## 已实现的蜕变关系

### 1. 匀速运动测试（Velocity-based Test）
**关系描述：** 如果模型以速度 v 沿某个方向运动 t 秒，位移应该是 v*t。

**实现状态：** ✅ 已实现 (`metamorphic_test_example`)

**测试流程：**
- 随机选择一个模型
- 获取初始位置
- 施加力使模型沿指定方向运动
- 等待指定时间
- 获取最终位置
- 验证位移是否符合预期

### 2. 状态回溯测试（Rewind Test）
**关系描述：** 在时间 t 记录状态，运行 Δt 秒后回溯到时间 t，状态应该与记录时完全一致。

**实现状态：** ✅ 已实现 (`metamorphic_test_rewind`)

**测试流程：**
- 在时间 t_a 记录所有模型状态
- 继续运行 t_b 秒
- 回溯到时间 t_a
- 比较回溯后的状态与记录的状态

## 建议实现的蜕变关系

### 3. 力的可加性测试（Force Additivity Test）
**关系描述：** 对同一模型同时施加多个力 F1, F2, ..., Fn，结果应该等于这些力的矢量和 F_total = F1 + F2 + ... + Fn。

**测试流程：**
1. 随机选择一个模型，获取初始位置
2. **测试 A：** 同时施加力 F1 和 F2，运行 t 秒，记录最终位置 P1
3. **测试 B：** 施加力 F_total = F1 + F2，运行 t 秒，记录最终位置 P2
4. 验证 P1 ≈ P2（在容差范围内）

**实现要点：**
- 需要能够同时施加多个力（可以通过多次调用 `func_apply_model_force` 或在一个命令中组合）
- 注意力的方向是矢量，需要矢量相加

**预期结果：** 如果物理引擎正确实现了力的叠加，两个测试的最终位置应该相同。

---

### 4. 力的比例性测试（Force Proportionality Test）
**关系描述：** 如果施加力 F 产生位移 d，施加 k*F 应该产生 k*d（在相同时间内，假设线性系统）。

**测试流程：**
1. 随机选择一个模型，获取初始位置
2. **测试 A：** 施加力 F，运行 t 秒，记录位移 d1
3. **测试 B：** 施加力 k*F（k 是比例因子，如 2.0），运行 t 秒，记录位移 d2
4. 验证 d2 ≈ k*d1（在容差范围内）

**实现要点：**
- 需要确保初始条件相同（位置、速度）
- 比例因子 k 应该选择合理的值（如 0.5, 2.0, 3.0）
- 注意：在非线性系统（如碰撞、摩擦）中，这个关系可能不成立

**预期结果：** 在理想线性系统中，位移应该与力成正比。

---

### 5. 时间缩放测试（Time Scaling Test）
**关系描述：** 如果改变 `real_time_factor`，物理行为应该按比例变化。例如，real_time_factor=2 时，模拟时间应该以 2 倍速度前进。

**测试流程：**
1. 随机选择一个模型，获取初始位置
2. **测试 A：** 设置 real_time_factor=1.0，施加力 F，运行真实时间 t_real，记录模拟时间 t_sim1 和最终位置 P1
3. **测试 B：** 设置 real_time_factor=2.0，施加力 F，运行真实时间 t_real，记录模拟时间 t_sim2 和最终位置 P2
4. 验证：
   - t_sim2 ≈ 2 * t_sim1
   - P2 ≈ P1（因为物理行为应该相同，只是时间流逝更快）

**实现要点：**
- 需要使用 `/world/<world_name>/set_physics` 服务来设置 `real_time_factor`
- 需要监控模拟时间（通过 `/stats` topic）
- 注意：real_time_factor 只影响时间流逝速度，不影响物理计算

**预期结果：** 模拟时间应该按比例变化，但物理行为（位置、速度）应该相同。

---

### 6. 重力反转测试（Gravity Reversal Test）
**关系描述：** 如果反转重力方向，模型在垂直方向的运动应该对称。

**测试流程：**
1. 随机选择一个模型，获取初始位置 (x0, y0, z0)
2. **测试 A：** 使用默认重力 g = (0, 0, -9.8)，运行 t 秒，记录最终位置 (x1, y1, z1)
3. **测试 B：** 设置重力 g_reverse = (0, 0, 9.8)，从相同初始位置开始，运行 t 秒，记录最终位置 (x2, y2, z2)
4. 验证：
   - x2 ≈ x1, y2 ≈ y1（水平方向应该相同）
   - z2 ≈ 2*z0 - z1（垂直方向应该对称）

**实现要点：**
- 需要使用 `/world/<world_name>/set_physics` 服务来设置重力
- 需要确保两次测试的初始条件完全相同
- 注意：如果模型有初始速度，需要考虑速度的影响

**预期结果：** 垂直方向的运动应该关于初始位置对称。

---

### 7. 位置对称性测试（Position Symmetry Test）
**关系描述：** 如果模型在位置 (x, y, z) 的行为是 A，在位置 (-x, -y, -z) 的行为应该是对称的（假设没有外部场）。

**测试流程：**
1. 随机选择一个模型
2. **测试 A：** 将模型放置在位置 (x, y, z)，施加力 F，运行 t 秒，记录最终位置 P1
3. **测试 B：** 将模型放置在位置 (-x, -y, -z)，施加力 -F（反向），运行 t 秒，记录最终位置 P2
4. 验证 P2 ≈ -P1（在容差范围内）

**实现要点：**
- 需要使用 `set_pose` 服务来设置模型位置
- 需要确保两次测试的其他条件相同（重力、摩擦等）
- 注意：如果世界中有非对称的场（如磁场、风），这个关系可能不成立

**预期结果：** 在对称的世界中，行为应该是对称的。

---

### 8. 质量缩放测试（Mass Scaling Test）
**关系描述：** 根据 F = ma，在相同力 F 下，质量 m 越大，加速度 a 越小。如果质量变为 k*m，加速度应该变为 a/k。

**测试流程：**
1. 随机选择一个模型，获取其质量 m1
2. **测试 A：** 使用原始质量 m1，施加力 F，运行 t 秒，记录位移 d1
3. **测试 B：** 将质量改为 m2 = k*m1（k 是比例因子），施加力 F，运行 t 秒，记录位移 d2
4. 验证 d2 ≈ d1/k（在容差范围内）

**实现要点：**
- 需要能够获取和修改模型的质量（可能需要通过 SDF 修改或使用服务）
- 注意：修改质量可能需要重新加载模型或使用特殊服务
- 比例因子 k 应该选择合理的值（如 0.5, 2.0）

**预期结果：** 在相同力下，质量越大，位移越小（成反比）。

---

### 9. 力的时间顺序测试（Force Time Order Test）
**关系描述：** 先施加力 F1 持续 t1，再施加力 F2 持续 t2，应该与先 F2 后 F1 的结果不同（验证时间顺序的重要性）。

**测试流程：**
1. 随机选择一个模型，获取初始位置
2. **测试 A：** 
   - 施加力 F1，运行 t1 秒
   - 施加力 F2，运行 t2 秒
   - 记录最终位置 P1
3. **测试 B：**
   - 施加力 F2，运行 t2 秒
   - 施加力 F1，运行 t1 秒
   - 记录最终位置 P2
4. 验证 P1 ≠ P2（除非 F1 = F2 或 t1 = t2）

**实现要点：**
- 需要能够按顺序施加不同的力
- 注意：这个测试验证的是时间顺序的重要性，结果应该不同

**预期结果：** 在一般情况下，P1 ≠ P2，因为速度是累积的。

---

### 10. 能量守恒测试（Energy Conservation Test）
**关系描述：** 在理想系统中（无摩擦、无碰撞），动能和势能之和应该守恒。

**测试流程：**
1. 随机选择一个模型，获取初始位置和速度
2. 计算初始总能量 E0 = KE0 + PE0
   - KE0 = 0.5 * m * v^2
   - PE0 = m * g * h
3. 运行模拟 t 秒（不施加外力）
4. 获取最终位置和速度
5. 计算最终总能量 E1 = KE1 + PE1
6. 验证 E1 ≈ E0（在容差范围内）

**实现要点：**
- 需要获取模型的速度（可能需要通过 topic 或服务）
- 需要获取模型的质量
- 需要计算动能和势能
- 注意：在有摩擦或碰撞的系统中，能量不守恒是正常的

**预期结果：** 在理想系统中，总能量应该守恒。

---

### 11. 碰撞对称性测试（Collision Symmetry Test）
**关系描述：** 两个模型碰撞，交换它们的初始位置和速度，结果应该对称。

**测试流程：**
1. 选择两个模型 A 和 B
2. **测试 A：**
   - 模型 A 在位置 P_A，速度 V_A
   - 模型 B 在位置 P_B，速度 V_B
   - 运行直到碰撞发生，记录碰撞后的状态
3. **测试 B：**
   - 模型 A 在位置 P_B，速度 V_B
   - 模型 B 在位置 P_A，速度 V_A
   - 运行直到碰撞发生，记录碰撞后的状态
4. 验证碰撞后的状态应该对称（交换模型标识）

**实现要点：**
- 需要能够设置模型的初始位置和速度
- 需要检测碰撞事件（可能需要订阅碰撞 topic）
- 需要记录碰撞前后的状态

**预期结果：** 碰撞结果应该对称。

---

### 12. 坐标系旋转不变性测试（Coordinate Rotation Invariance Test）
**关系描述：** 旋转坐标系，物理行为应该保持一致。

**测试流程：**
1. 随机选择一个模型，获取初始位置 P0
2. **测试 A：** 在原始坐标系中，施加力 F，运行 t 秒，记录最终位置 P1
3. **测试 B：** 旋转坐标系（例如绕 Z 轴旋转 90 度），在旋转后的坐标系中：
   - 初始位置变为 P0_rot
   - 施加力 F_rot（相应旋转），运行 t 秒，记录最终位置 P1_rot
4. 将 P1_rot 旋转回原始坐标系，验证与 P1 相同

**实现要点：**
- 需要理解坐标变换
- 需要能够旋转力和位置
- 注意：这主要测试物理引擎的坐标系无关性

**预期结果：** 物理行为应该与坐标系选择无关。

## 实现优先级建议

### 高优先级（容易实现，价值高）
1. ✅ **匀速运动测试** - 已实现
2. ✅ **状态回溯测试** - 已实现
3. **力的可加性测试** - 相对简单，验证基本物理定律
4. **力的比例性测试** - 相对简单，验证线性系统

### 中优先级（需要额外功能，但可行）
5. **时间缩放测试** - 需要设置 real_time_factor
6. **重力反转测试** - 需要设置重力
7. **位置对称性测试** - 需要设置模型位置

### 低优先级（实现复杂或需要特殊条件）
8. **质量缩放测试** - 可能需要修改 SDF 或使用特殊服务
9. **力的时间顺序测试** - 相对简单，但验证的是"不同"而非"相同"
10. **能量守恒测试** - 需要获取速度信息
11. **碰撞对称性测试** - 需要检测碰撞事件
12. **坐标系旋转不变性测试** - 需要坐标变换知识

---

## 实现注意事项

1. **初始条件一致性：** 确保每次测试的初始条件（位置、速度、质量等）相同
2. **容差设置：** 由于数值误差，需要设置合理的容差
3. **非线性效应：** 某些关系只在理想线性系统中成立，需要考虑摩擦、碰撞等非线性效应
4. **时间同步：** 确保使用模拟时间而非真实时间
5. **状态重置：** 在测试之间可能需要重置模拟状态
